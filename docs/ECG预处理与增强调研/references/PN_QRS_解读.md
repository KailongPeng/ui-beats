## 0. QRS 波群检测：完整背景知识

### 0.1 什么是 QRS 波群？

心脏每跳动一次，会产生一系列电信号，在心电图上形成一个固定的波形模式：

```
        R
        /\
       /  \
      /    \
  P  /      \  T
 /\ /    S   \/\
/  Q
```

- **P 波**：心房收缩（心房除极）
- **QRS 波群**：心室收缩（心室除极）——这是整个心电图中**振幅最大、最尖锐**的部分
  - Q 波：第一个向下的小偏转
  - R 波：向上的大尖峰（**整个心电图最醒目的点**）
  - S 波：R 波之后向下的偏转
- **T 波**：心室恢复（心室复极）

QRS 波群的持续时间通常只有 **60-100 毫秒**，但它是定位每一次心跳的"锚点"。

### 0.2 为什么 QRS 检测是"第一步"？

几乎所有心电分析都依赖于先找到 QRS 波（具体是 R 峰）的位置：

```
应用场景                     依赖 QRS 的原因
─────────────────────────────────────────────────
心率计算                  → 心率 = 60 / (相邻R峰间隔秒数)
心率变异性 (HRV)          → 分析 R-R 间期的波动规律
心律失常检测              → 判断节律是否规则、是否有早搏
ST 段分析（心肌缺血）     → ST 段定义为"S波结束到T波开始"，需先知道 S 在哪
QT 间期测量（药物安全性） → QT = Q波起点到T波终点，需先定位 Q
心电信号分割              → 把连续信号切成"一拍一拍"，每拍以 R 峰为中心
```

**如果 QRS 检测出错，后面所有分析都会跟着错。** 这就是为什么它是心电分析流水线的"地基"。

### 0.3 QRS 检测是自动化的吗？

**是的，QRS 检测在现代已经高度自动化。**

历史演进：

| 时代 | 方式 | 说明 |
|---|---|---|
| 1960s 以前 | 完全人工 | 医生用尺子在纸质心电图上量 |
| 1970s-1980s | 半自动 | 计算机辅助，但需要人工确认 |
| 1985 至今 | 全自动 | Pan-Tompkins 算法发表后成为标配 |
| 2018 至今 | 深度学习 | CNN/RNN/Transformer 端到端检测 |

如今，从医院的心电监护仪、Holter（动态心电记录仪）到 Apple Watch，**所有设备都内置了自动 QRS 检测算法**。医生看到的"心率 72 次/分"就是自动检测的结果。

### 0.4 传统 QRS 检测技术

#### (1) Pan-Tompkins 算法（1985）—— 里程碑式的经典

这是引用量最高的 QRS 检测算法（原始论文被引超过 10000 次），至今仍在大量设备中使用。

**核心思路**：通过一系列信号处理步骤，逐步放大 QRS 波、抑制其他成分：

```
原始 ECG
  │
  ▼
带通滤波 (5-15 Hz)          ← 只保留 QRS 频段，滤掉 P/T 波和高频噪声
  │
  ▼
微分 (求导)                  ← 突出 QRS 的陡峭上升沿
  │
  ▼
平方                         ← 让大值更大、小值更小，进一步拉开差距
  │
  ▼
滑动窗口积分 (150ms窗)      ← 平滑，得到一个"能量包络"
  │
  ▼
自适应阈值判决               ← 超过阈值的峰就是 R 峰
  │
  ▼
回溯搜索                     ← 如果间隔太长没检到，降低阈值重新找
```

**优点**：简单、快速、计算量极小（嵌入式芯片都能跑）
**缺点**：固定的滤波器参数对形态异常的 QRS（如宽 QRS、室性早搏）效果差；对低信噪比信号容易误检

#### (2) 基于小波变换的方法

小波变换可以同时分析信号的**时间位置**和**频率成分**，天然适合检测 QRS 这种"短时、宽频"的事件。

**代表方法**：
- **Li et al. (1995)**：用二进小波变换，在多个尺度上检测 QRS 的模极大值对
- **Martinez et al. (2004)**：基于小波的完整 ECG 波形界定（不仅检 QRS，还标注 P/T 波的起止点）

**通俗理解**：
> Pan-Tompkins 像用一个固定放大镜看信号，小波变换则像同时用多个不同倍率的放大镜看——既能看到整体趋势（低频），也能捕捉细节尖峰（高频）。

**优点**：对不同形态的 QRS 适应性更好
**缺点**：计算量比 Pan-Tompkins 大；小波基函数的选择需要经验

#### (3) 基于匹配滤波 / 模板匹配的方法

**思路**：预先存一个"标准 QRS 模板"，让它在信号上滑动，计算相似度，相似度最高的位置就是 QRS。

**优点**：直觉简单
**缺点**：QRS 形态因人而异，一个模板不能通吃；对噪声敏感

#### (4) 基于数学形态学的方法

利用膨胀、腐蚀等数学形态学运算来提取 QRS 波的结构特征。

**通俗理解**：
> 把心电信号看成一条"地形线"，用不同大小的"探针"去刮——探针比 QRS 窄时，QRS 尖峰会保留；比 QRS 宽时，尖峰会被削平。两者相减就能提取出 QRS。

#### (5) 其他传统方法

| 方法 | 核心思路 |
|---|---|
| 希尔伯特变换 | 利用信号瞬时相位/频率的突变来检测 QRS |
| 经验模态分解 (EMD) | 把信号分解为多个固有模态函数，在特定模态中检测 QRS |
| 自适应滤波 | 用参考信号消除噪声后再检测 |
| 基于能量的方法 | 计算短时能量，QRS 区间能量显著高于其他区间 |

### 0.5 现代（深度学习）QRS 检测技术

#### 为什么需要深度学习？

传统方法的局限：
1. **依赖人工设计特征**：滤波器参数、阈值都需要专家调
2. **泛化能力有限**：在一个数据集上调好的参数，换个数据集可能不行
3. **对异常形态 QRS 力不从心**：宽 QRS、碎裂 QRS、室颤时的 QRS 都很难检测
4. **可穿戴场景噪声太大**：传统方法假设信号"基本干净"

深度学习可以**直接从原始信号中自动学习特征**，不需要手动设计。

#### (1) CNN（卷积神经网络）方法

**代表**：U-Net 变体、多分支 CNN（本论文 PN-QRS 的编码器就是这类）

```
原始 ECG (1D 信号)
  │
  ▼
1D 卷积层 × N              ← 自动学习滤波器（替代手工设计的带通滤波）
  │
  ▼
逐帧分类                    ← 每个时间点输出"是QRS / 不是QRS"的概率
  │
  ▼
后处理找峰                   ← 在概率曲线上找局部最大值 → R 峰位置
```

**优点**：端到端学习，无需手动调参；对各种 QRS 形态泛化好
**代表工作**：Cai & Hu (2020)、He et al. (2020) 的 U-Net+BiLSTM

#### (2) RNN / LSTM 方法

利用循环神经网络捕捉心电信号的**时序依赖**（上一个心跳和下一个心跳的间隔应该大致稳定）。

**优点**：能利用上下文信息（如果某个位置"该出现"心跳了但没检到，模型会倾向于降低阈值去找）
**缺点**：训练速度慢，长序列容易梯度消失

#### (3) Transformer / 注意力机制方法

利用自注意力机制捕捉信号中的**长程依赖**。

**优点**：能"看到"整个 10 秒甚至更长片段的全局模式
**缺点**：计算量大，不太适合嵌入式实时场景

#### (4) 语义分割框架

把 QRS 检测转化为**时间序列语义分割**问题：

```
输入：[采样点1, 采样点2, ..., 采样点N]
输出：[非QRS,  非QRS, ..., QRS, QRS, QRS, ..., 非QRS, ...]
```

每个采样点都被分类为"QRS 区间"或"非 QRS 区间"，然后在 QRS 区间中找峰值就是 R 峰。

**PN-QRS 就属于这一类**——它的骨架网络 (temporal-framing architecture) 本质就是一个语义分割模型。

#### (5) 不确定性感知方法（本论文所属类别）

| 方法 | 原理 | 额外开销 |
|---|---|---|
| **MC-Dropout** | 推理时保留 Dropout，多次前向传播取方差 | 需要 ~30 次推理 |
| **Deep Ensemble** | 训练 5 个独立模型，看预测是否一致 | 5 倍训练 + 5 倍推理 |
| **PN-QRS（本文）** | 正负变换，1 次推理同时得到 3 组输出 | 仅增加 2 个小 MLP |

PN-QRS 的核心竞争力就在于：**用极低的额外开销实现了不确定性估计**。

### 0.6 传统 vs 深度学习：一览对比

| 维度 | 传统方法 | 深度学习方法 |
|---|---|---|
| **代表** | Pan-Tompkins, 小波变换 | CNN, U-Net, LSTM, Transformer |
| **特征提取** | 人工设计（滤波、微分、阈值） | 自动学习 |
| **调参** | 需要领域专家 | 需要数据 + GPU |
| **计算量** | 极低（可在MCU上运行） | 较高（需要 GPU 或 NPU） |
| **泛化能力** | 弱（参数针对特定数据集） | 强（大数据训练后泛化好） |
| **对异常QRS** | 容易漏检 | 表现好 |
| **对噪声鲁棒性** | 中等 | 强 |
| **可解释性** | 强（每步都有物理意义） | 弱（黑箱） |
| **实时性** | 极强 | 中等（取决于模型大小） |
| **部署场景** | 嵌入式设备、监护仪 | 云端/边缘计算 |

**实际应用中的趋势**：
- **医院监护仪 / Holter**：仍然以传统方法为主（Pan-Tompkins 变体），因为需要实时、低功耗
- **云端分析平台**：逐渐转向深度学习（可以跑更大的模型）
- **可穿戴设备**：传统方法做实时检测 + 深度学习做离线精分析（混合方案）
- **科研前沿**：深度学习方法已全面超越传统方法的精度上限

### 0.7 常用评测数据集与指标

#### 数据集

| 数据集 | 导联数 | 记录数 | 场景 |
|---|---|---|---|
| **MIT-BIH** | 2 | 48 条 30 分钟 | 心律失常（最经典的benchmark） |
| **INCART** | 12 | 75 条 30 分钟 | 12 导联动态心电 |
| **QTDB** | 2 | 105 条 15 分钟 | QT 间期研究 |
| **LUDB** | 12 | 200 条 10 秒 | 12 导联，详细波形标注 |
| **CPSC2019** | 1 | ~2000 条 10 秒 | 单导联，中国生理信号挑战赛 |

#### 评价指标

- **Se（灵敏度）**：真实心跳中被正确检测到的比例 → 漏检越少越好
- **P+（阳性预测值）**：检测出的心跳中真正是心跳的比例 → 误检越少越好
- **F1**：Se 和 P+ 的调和平均 → 综合指标
- **容差窗口**：通常允许检测位置与标注位置偏差 **±150ms**（即约 37 个采样点 @250Hz）

## 一句话总结

PN-QRS 是一个能**感知自身"不确定性"的 QRS 波检测器**——它不仅能找心跳（QRS 波），还能告诉你"这段信号到底是不是心电"以及"这个心跳是不是可疑的伪差"。

## 1. 这篇论文要解决什么问题？

### 背景：可穿戴心电的痛点

在医院里做心电图，信号很干净。但可穿戴设备（手表、贴片）采集的长程心电充满了各种干扰：

- **运动伪差**：走路、跑步时肌肉颤动产生的电信号
- **基线漂移**：呼吸、电极接触不良导致信号整体上下飘
- **交流干扰**：50/60Hz 电源干扰
- **随机噪声**：各种环境电磁干扰

### 核心矛盾

QRS 波群检测是心电分析的**第一步**（后续的心率计算、心律失常诊断都依赖它）。但在可穿戴场景下：

1. **噪声太大时**，一段信号可能根本不是有效心电（比如电极脱落），但检测器仍然会"硬找"出一些假的 QRS 波
2. **伪差长得像 QRS 波**，检测器会把它当成真心跳（如 Fig.1 中的 QRS-like artifact）
3. 传统方法只给出"是/否"的结果，**不告诉你它有多确信**

这就像一个考生在考试中瞎蒙了答案但不标注"我不确定"——老师无法知道哪些答案是可靠的。

## 2. 两种不确定性：深度直觉解释

这是理解整篇论文的最关键前提。论文把模型"拿不准"分成了两种根本不同的原因，每种原因对应不同的处理方式。

### 2.0 极简通俗版：用一个生活场景读懂两种不确定性

> 这部分不涉及任何公式或机器学习术语，适合第一次接触这个概念的读者。

想象你是一位法医鉴定专家，有人拿来一张纸，问你："这上面有没有指纹？"

**情形 A：拿来的是一张彩色广告传单**

你盯着它看了半天，越看越困惑——你从没做过彩色印刷纸的指纹鉴定，你受的训练全是白纸。你不是在说传单上的图案模糊，你是说**你根本不具备鉴定这种材料的能力**。

→ 这是**认知不确定性**：问题出在"我"，是能力边界的问题。

**情形 B：拿来的是一张白纸，但印章覆盖了可能有指纹的区域**

你非常擅长白纸指纹鉴定，工具齐全，经验丰富。但这张纸上偏偏有一枚大红印章压在最关键的位置上，指纹被遮住了一半，真假难辨。**不是你不行，是这份样本本身就残缺模糊**。

→ 这是**偶然不确定性**：问题出在"材料"，是数据本身的问题。

---

**对应到心电信号：**

```
输入一段信号
        │
        ▼
 [第一道门] 认知不确定性判断
        │
        ├── 高 → "这不是心电图，是噪声/脉搏波/心音"
        │         → 丢弃整段，不再分析
        │
        └── 低 → "这确实是心电图，我有能力处理"
                  │
                  ▼
         [第二道门] 偶然不确定性判断（逐心拍）
                  │
                  ├── 高 → "这个心跳形态太模糊，可能是伪差"
                  │         → 标记为可疑，请人工复核
                  │
                  └── 低 → "这个心跳清晰可信"
                            → 正常输出 R-peak 位置
```

**一句话区分**：

| | 认知不确定性 | 偶然不确定性 |
|---|---|---|
| **症结** | 模型没见过这类数据 | 数据本身就模糊 |
| **类比** | 法医从没做过彩色纸鉴定 | 指纹被印章遮住了 |
| **处理** | 整段丢弃 | 逐拍标记 |
| **能改善吗** | 能——让模型见更多类型的数据 | 不能——材料本身如此 |

### 2.1 认知不确定性（Epistemic Uncertainty）—— "我没学过，所以我不懂"

#### 定义

认知不确定性来自**模型自身知识的不足**——训练数据不够多、不够全面，导致模型对某些类型的输入"没见过世面"。

#### 核心特征：可以通过增加训练数据来消除

如果给模型看更多样的数据，认知不确定性就会降低。它是模型的"无知"。

#### 多层类比帮助理解

**类比 1：新手医生**

一个刚毕业的心内科医生，在学校里只看过正常窦性心律和常见心律失常的心电图。

- 你给他看一份**正常心电图** → 他立刻认出来 → 认知不确定性**低**
- 你给他看一份**房颤心电图** → 他也学过 → 认知不确定性**低**
- 你给他看一份**PPG 信号**（脉搏波，不是心电图，但也有周期性波形）→ 他从来没见过这种东西，完全不知道该怎么分析 → 认知不确定性**极高**
- 你给他看一份**心音图**（心脏声音信号）→ 他也没学过 → 认知不确定性**极高**

关键点：这些他不懂的东西，并不是因为信号本身模糊，而是**因为他的教育不够全面**。如果他去进修学习了 PPG 和心音图，下次遇到就不会困惑了。

**类比 2：考试**

- 考试范围内的题目 → 你学过，有信心答 → 认知不确定性低
- 完全超纲的题目 → 你根本没学过这个知识点，连题目在问什么都看不懂 → 认知不确定性高

注意：超纲题的难度不在于题目本身模糊，而在于**你的知识覆盖不到这个范围**。

#### 在 PN-QRS 中的具体含义

PN-QRS 的模型是在正常 ECG 信号上训练的（CPSC2019 数据集）。所以：

| 输入类型 | 模型见过吗？ | 认知不确定性 | 应该怎么处理 |
|---|---|---|---|
| 干净的正常 ECG | 见过大量 | **低** | 正常检测 QRS |
| 有噪声但可辨认的 ECG | 见过一些 | **中低** | 正常检测，可能稍有犹豫 |
| 纯随机噪声（电极脱落）| **从没见过** | **极高** | 应该拒绝处理——这不是 ECG！ |
| PPG 脉搏波信号 | **从没见过** | **极高** | 应该拒绝处理——这不是 ECG！ |
| 心音信号 | **从没见过** | **极高** | 应该拒绝处理——这不是 ECG！ |

所以认知不确定性本质上是一个 **"这是不是 ECG？"的信号质量看门狗**。它回答的核心问题是：**"这段信号在不在我学过的范围内？"**

在机器学习术语中，这对应于 **OoD（Out-of-Distribution，分布外）检测**——判断输入是否属于训练数据所涵盖的分布。

### 2.2 偶然不确定性（Aleatoric Uncertainty）—— "数据本身就模糊，谁来看都会犹豫"

#### 定义

偶然不确定性来自**数据本身的固有噪声和歧义**——即使模型是全知全能的，数据本身就不够清晰，也无法给出确定的答案。

#### 核心特征：无法通过增加训练数据来消除

因为问题不在模型，而在数据本身。这是数据的"天生模糊"。

#### 多层类比帮助理解

**类比 1：模糊的照片**

想象你在辨认一系列人脸照片：

- 一张**清晰的照片** → 你能立刻认出这是谁 → 偶然不确定性**低**
- 一张**严重模糊、曝光过度的照片** → 你觉得"好像是张三，但也可能是李四" → 偶然不确定性**高**

注意：这张模糊照片的问题不在于你的视力或认知能力（你是人脸识别专家），而在于**照片本身质量太差，信息量不足以做出确定判断**。即使你再怎么训练自己的眼睛，面对这张模糊照片你仍然会犹豫。

**类比 2：考试中措辞模糊的题目**

- "心房颤动的 ECG 特征是什么？" → 问题清晰明确 → 偶然不确定性低
- "请判断这份心电图是否正常（附了一份质量极差、干扰很多的心电图）" → 不是你不会，而是**题目给的材料本身就看不清** → 偶然不确定性高

**类比 3：掷骰子**

你知道骰子是公正的，六面等概率。下一次掷出几？你无法确定——不是因为你对骰子不了解（认知不确定性为零），而是因为**掷骰子本身就是随机的**（偶然不确定性是固有的）。

#### 在 PN-QRS 中的具体含义

即使一段信号确实是 ECG（认知不确定性低），里面的某些心跳仍然可能是模糊的：

| 心跳类型 | 看起来像什么 | 偶然不确定性 | 实际情况 |
|---|---|---|---|
| 清晰的正常心跳 | 教科书般的 QRS 波形 | **低** | 百分百是心跳 |
| 清晰的室性早搏 | QRS 很宽但形态明确 | **低** | 百分百是心跳（只是异常心跳） |
| 运动伪差中的一个尖峰 | 有点像 QRS，但又不太像 | **高** | 不确定——可能是心跳也可能是伪差 |
| 从另一条 ECG 记录中"串"进来的 QRS | 长得完全像 QRS（因为它本来就是另一个心跳）但在时间上不符合当前心律节奏 | **高** | 是伪差（QRS-like artifact） |

所以偶然不确定性本质上是一个 **"这个心跳可信吗？"的逐拍质量标记**。它回答的核心问题是：**"就算这确实是 ECG 信号，这个具体的心跳本身是清晰还是模糊的？"**

### 2.3 两种不确定性的关键区别——一张图看懂

```
                    认知不确定性 低                    认知不确定性 高
                 （模型见过这类数据）              （模型没见过这类数据）
               ┌─────────────────────┐      ┌─────────────────────┐
偶然不确定性 低  │                     │      │                     │
（数据本身清晰）│  干净 ECG，清晰心跳   │      │    不存在这个组合*    │
               │  → 正常检测，高可信    │      │                     │
               │  [完美情况]           │      │                     │
               ├─────────────────────┤      ├─────────────────────┤
偶然不确定性 高  │                     │      │                     │
（数据本身模糊）│  是 ECG，但心跳模糊   │      │  纯噪声 / PPG / 心音  │
               │  → 标记为"可疑心跳"   │      │  → 直接拒绝，不是ECG  │
               │  [需要人工审核]       │      │  [丢弃这段信号]       │
               └─────────────────────┘      └─────────────────────┘

* 注：如果模型完全没见过一类数据（认知不确定性极高），
  那么偶然不确定性的估计本身就不可靠了，所以这个组合没有实际意义。
  这就是为什么 PN-QRS 先检查认知不确定性（第一道门），
  通过后才去看偶然不确定性（第二道门）。
```

### 2.4 为什么一定要区分这两种不确定性？

因为**处理方式完全不同**：

| | 认知不确定性高 | 偶然不确定性高 |
|---|---|---|
| **含义** | 这段数据不在模型的能力范围内 | 这个心跳本身就模糊 |
| **正确处理** | 整段丢弃（不是 ECG，别分析了） | 标记该心跳为可疑，但保留整段信号 |
| **错误处理** | 如果不丢弃 → 检测出一堆假心跳 → 心率计算全错 | 如果不标记 → 伪差被当成真心跳 → 心律失常误诊 |
| **能否改善** | 能——给模型看更多类型的数据 | 不能——数据本身就是这么模糊 |

如果不区分，只用一个笼统的"置信度"，就无法区分"不是 ECG"和"是 ECG 但心跳模糊"，这两种情况的处理策略截然不同。

## 3. 正负变换：深度直觉解释

这是论文名字 **P**ositive-**N**egative 的来源，也是最核心的技术创新。

### 3.1 先理解"特征空间"

在正式解释正负变换之前，需要先理解一个前置概念：**特征空间**。

当一段 ECG 信号（2500 个采样点 @250Hz × 10 秒）经过编码器后，每一帧（16ms）都被压缩成一个高维向量 z_τ。你可以把这个向量想象成这一帧信号的"身份证"——一串数字，编码了这帧信号的所有特征。

为了便于理解，假设这个向量是二维的（实际上是高维的，但道理一样）。那么所有帧的特征就可以画在一个二维平面上：

```
特征空间（简化为 2D）

  ^  特征维度2
  |
  |     ●●●           ○○○
  |    ●●●●●         ○○○○○
  |     ●●●           ○○○          ● = QRS 帧的特征
  |                                 ○ = 非 QRS 帧的特征
  |         △△△△
  |        △△△△△△
  |         △△△△                   △ = 噪声帧的特征
  |                                    （模型没见过的 OoD 数据）
  └──────────────────────> 特征维度1
```

在训练好的模型中：
- QRS 帧的特征（●）聚集在一片区域
- 非 QRS 帧的特征（○）聚集在另一片区域
- 噪声/非 ECG 帧的特征（△）可能落在任何地方——因为模型没见过这类数据，编码器不知道该把它们放哪

### 3.2 正负变换到底做了什么？—— "分身术"

**正变换 φ⁺** 和 **负变换 φ⁻** 各是一个小型 MLP（多层感知器），它们对特征向量 z 做一个"推动"：

- **φ⁺（正变换）**：把 z 往 **QRS 帧的聚集中心** 方向推 → 得到 z⁺
- **φ⁻（负变换）**：把 z 往 **非 QRS 帧的聚集中心** 方向推 → 得到 z⁻

```
特征空间中的正负变换（单帧示意）

                    QRS 帧的聚集中心 (r_p)
                         ★
                        ↗
               z⁺ = φ⁺(z)
                      ↗
  原始特征 z ——→ ●
                      ↘
               z⁻ = φ⁻(z)
                        ↘
                         ★
                    非QRS帧的聚集中心 (r_n)
```

**通俗理解：正负变换就是"分身术"**

想象你面对一段信号，不确定它是 QRS 还是非 QRS。正负变换做的事情是：

1. **正变换**：创造一个"乐观分身"——假设这帧就是 QRS，把它的特征往 QRS 的方向推一把，看看推过去之后解码器会怎么判断
2. **负变换**：创造一个"悲观分身"——假设这帧不是 QRS，把它的特征往非 QRS 的方向推一把，看看推过去之后解码器会怎么判断

然后把原始特征和两个分身**都送进同一个解码器**，得到三个预测结果。

### 3.3 具体例子：正负变换在不同场景下的表现

为了真正理解正负变换的妙处，我们看三个具体场景：

#### 场景 A：输入是一个**明确的 QRS 帧**（来自干净 ECG 的 R 峰附近）

```
          QRS 区域
           ★ (r_p)
          ↗
    z⁺ ← φ⁺(z)      ← z⁺ 推向 QRS 中心，和 z 本来就在差不多的地方
        ↗
z ──→ ● (本来就在 QRS 区域附近)
        ↘
    z⁻ ← φ⁻(z)      ← z⁻ 推向非QRS中心，推离了原始位置
          ↘
           ★ (r_n)
          非QRS 区域

解码器的三个输出：
  ŷ  (原始)  = 0.95 (很可能是 QRS)
  ŷ⁺ (正变换) = 0.98 (更确信是 QRS —— 往 QRS 方向推了，当然更像 QRS)
  ŷ⁻ (负变换) = 0.80 (虽然往非 QRS 方向推了，但解码器仍然说"这还是像 QRS")

关键观察：ŷ⁺ 和 ŷ⁻ 虽然有差异，但差异不大（0.98 vs 0.80）。
→ 两个分身意见相近 → 认知不确定性低 → "模型很确定这是什么"
→ 且正变换输出 (0.98) 和负变换输出 (0.80) 都偏向"是 QRS"→ 偶然不确定性低
→ 结论：确定的 QRS，高可信。
```

#### 场景 B：输入是**纯随机噪声**（电极脱落，不是 ECG）

```
          QRS 区域
           ★ (r_p)
          ↗
    z⁺ ← φ⁺(z)      ← 推向 QRS 中心
        ↗
z ──→ △ (噪声帧的特征在一个"奇怪"的位置，模型没见过)
        ↘
    z⁻ ← φ⁻(z)      ← 推向非 QRS 中心
          ↘
           ★ (r_n)
          非QRS 区域

解码器的三个输出：
  ŷ  (原始)  = 0.45 (模型很困惑)
  ŷ⁺ (正变换) = 0.88 (推向 QRS 后，解码器被"欺骗"说像 QRS)
  ŷ⁻ (负变换) = 0.12 (推向非 QRS 后，解码器说不像 QRS)

关键观察：ŷ⁺ 和 ŷ⁻ 差异巨大（0.88 vs 0.12）！
→ 两个分身意见完全相反 → 认知不确定性极高 → "模型完全不知道这是什么"
→ 结论：这段不是 ECG，应该直接丢弃。
```

**这就是认知不确定性的估计原理：两个分身越"吵架"（意见分歧越大），说明模型越不懂这个输入。**

为什么对 OoD 数据两个分身会大吵一架？因为对于模型没见过的数据，编码器产出的特征 z 落在一个"没有被训练覆盖"的空白地带。正变换把它推向 QRS 区域，解码器就说"像 QRS"；负变换把它推向非 QRS 区域，解码器就说"不像 QRS"。推到哪个方向就像哪个方向——这说明原始特征 z 本身不携带有意义的信息，模型对它完全没有"定见"。

相比之下，对于模型见过的正常 ECG，编码器产出的特征 z 本身就有强烈的倾向性（要么接近 QRS 区域，要么接近非 QRS 区域），所以正负变换怎么推都无法根本改变解码器的判断——"分身"之间大致一致。

#### 场景 C：输入是**真实 ECG 中一个模糊心跳**（QRS-like artifact）

```
          QRS 区域
           ★ (r_p)
          ↗
    z⁺ ← φ⁺(z)      ← 推向 QRS 中心
        ↗
z ──→ ● (在 QRS 和非 QRS 区域的边界地带——因为这个心跳本身就模棱两可)
        ↘
    z⁻ ← φ⁻(z)      ← 推向非 QRS 中心
          ↘
           ★ (r_n)
          非QRS 区域

解码器的三个输出：
  ŷ  (原始)  = 0.55 (模型犹豫不决)
  ŷ⁺ (正变换) = 0.75 (往 QRS 推后说"大概是 QRS")
  ŷ⁻ (负变换) = 0.35 (往非 QRS 推后说"大概不是 QRS")

关键观察：
→ ŷ⁺ 和 ŷ⁻ 有分歧但不极端 (0.75 vs 0.35) → 认知不确定性中低
  （模型见过这类数据——它确实是 ECG，只是这个心跳不太正常）
→ 但正变换输出 (0.75) 和负变换输出 (0.35) 本身的熵都较高
  （不像场景 A 中 0.98/0.80 那么确定）→ 偶然不确定性高
→ 结论：这是 ECG（通过认知不确定性检查），但这个心跳可疑（偶然不确定性高），标记为 ambiguous beat。
```

### 3.4 正负变换 vs 其他方案的对比

为什么不用其他更简单的方法来估计不确定性？

| 方法 | 做法 | 问题 |
|---|---|---|
| **直接看解码器输出概率** | 如果输出 0.5 就说不确定 | 深度学习模型会过度自信——对 OoD 数据也可能输出 0.99。无法区分两种不确定性 |
| **MC-Dropout** | 推理时随机关闭神经元，跑 30 次，看结果方差 | 需要 30 次前向传播，太慢！QRS 检测是心电分析的第一步，需要快 |
| **Deep Ensemble** | 训练 5 个完全独立的模型，看它们是否一致 | 需要 5 倍训练和 5 倍推理成本 |
| **PN-QRS 正负变换** | 训练 2 个小 MLP，跑 1 次推理得到 3 组输出 | 额外开销极小，且能同时分离认知和偶然不确定性 |

**正负变换的精妙之处**在于：它用一个非常廉价的操作（2 个小 MLP）模拟了 ensemble 的效果——通过人为制造两个有偏的视角（乐观分身和悲观分身），观察它们在同一个输入上的分歧程度，从而推断模型对该输入的确定程度。

### 3.5 正负变换的训练目标——通俗解释

φ⁺ 和 φ⁻ 怎么训练出来的？训练目标非常直觉：

**φ⁺（正变换）的训练目标**：
> "对于所有 QRS 帧，把它们的特征推向 QRS 帧群体的'平均长相'。"

具体来说：
1. 在训练集中收集所有被标注为 QRS 的帧的特征向量，计算它们的平均值 → 得到 QRS 的"典型代表" r_p
2. 训练 φ⁺，使得 φ⁺(z) 对于 QRS 帧要接近 r_p（用余弦相似度度量）
3. 训练时用 **MASK** 只关注 QRS 帧——对于非 QRS 帧，φ⁺ 不受约束（随便推到哪）

**φ⁻（负变换）的训练目标**：
> "对于所有非 QRS 帧，把它们的特征推向非 QRS 帧群体的'平均长相'。"

对称地：
1. 收集所有非 QRS 帧的特征向量，计算平均值 → 得到非 QRS 的"典型代表" r_n
2. 训练 φ⁻，使得 φ⁻(z) 对于非 QRS 帧要接近 r_n
3. 训练时用 MASK 只关注非 QRS 帧

```
训练示意（用 MASK 实现"各管各的"）：

φ⁺ 的训练：
  QRS 帧:     φ⁺(z) ——→ 靠近 r_p ✅  （受约束）
  非QRS帧:    φ⁺(z) ——→ 随意       （不约束，MASK 掉）

φ⁻ 的训练：
  QRS 帧:     φ⁻(z) ——→ 随意       （不约束，MASK 掉）
  非QRS帧:    φ⁻(z) ——→ 靠近 r_n ✅  （受约束）
```

**直觉理解**：φ⁺ 被训练成"对 QRS 帧特别敏感"，φ⁻ 被训练成"对非 QRS 帧特别敏感"。每个变换都是该类别的"专家"。

为什么用余弦距离（不是欧氏距离）？因为在高维空间中，余弦距离衡量的是方向的相似性，不受向量长度的影响，避免特征坍缩（所有特征都被推到同一个点）。

训练中 r_p 和 r_n 用 **Monte-Carlo 近似** 估计——每个 training epoch 中，用当前 mini-batch 中所有样本的 QRS/非 QRS 帧特征的平均值来近似全局平均值。

## 4. 不确定性的数学计算——直觉版

### 4.1 认知不确定性 U_E 的计算

认知不确定性衡量的是"两个分身的分歧程度"。具体用 **KL 散度** 计算：

```
U_E = (1/T) × Σ [ ŷ⁺ × log(ŷ⁺/ŷ) + ŷ⁻ × log(ŷ⁻/ŷ) ]
```

**直觉理解**：
- KL 散度衡量两个概率分布的差异
- ŷ⁺ 和 ŷ⁻ 是正/负变换后的预测概率
- ŷ 是原始预测概率
- 如果 ŷ⁺ ≈ ŷ ≈ ŷ⁻（三者差不多）→ KL 散度接近 0 → U_E 低 → "模型很确定"
- 如果 ŷ⁺ 远离 ŷ 或 ŷ⁻ 远离 ŷ → KL 散度大 → U_E 高 → "模型很困惑"

**阈值**：论文设定 α = 0.1，如果 U_E > 0.1，整段信号被判定为"非 ECG"。

**实验验证（Fig.5）**：论文画了正常 ECG 和四类非 ECG 信号（高斯噪声、心音、PPG、真实噪声）的 U_E 分布直方图。结果显示两者几乎没有重叠——说明认知不确定性确实能有效区分 ECG 和非 ECG。

### 4.2 偶然不确定性 U_A 的计算

偶然不确定性衡量的是"正负变换后预测结果本身的模糊程度"。具体用**二元交叉熵**（熵的变体）计算：

```
U_A = -[ ŷ⁺ × log(ŷ⁺) + (1 - ŷ⁻) × log(1 - ŷ⁻) ]
```

但更本质的理解是——这其实就是**正变换预测和负变换预测各自的"纠结程度"的综合**。

**直觉理解**：
- 对于确定的 QRS 帧：ŷ⁺ ≈ 0.99（正变换说"肯定是 QRS"），ŷ⁻ ≈ 0.80（负变换也觉得"大概是 QRS"）→ 两者都不纠结 → 熵低 → U_A 低
- 对于模糊的心跳：ŷ⁺ ≈ 0.65（正变换说"好像是 QRS？"），ŷ⁻ ≈ 0.45（负变换说"好像不是？"）→ 两者都很纠结 → 熵高 → U_A 高

**阈值**：论文设定 β = 0.12，对每个检测到的 R 峰，取其前后共 7 帧（112ms）的平均 U_A，如果 > 0.12，标记为"可疑心跳"。

**实验验证（Fig.7）**：论文画了确定心跳和模糊心跳的 U_A 分布直方图。确定心跳的 U_A 集中在极低值附近（接近 0），而模糊心跳的 U_A 明显向右偏移——两者可以用 β = 0.12 有效区分。

## 5. 完整推理流程——把所有概念串起来

```
输入：一段 10 秒的 ECG 片段（可能包含 L 个导联）

═══════════════════════════════════════════════════
第 0 步：对每个导联独立处理
═══════════════════════════════════════════════════
  对第 l 个导联：
    x_l → 编码器 f_θ → 特征 z_l → 625 帧

═══════════════════════════════════════════════════
第 1 步：正负变换 + 解码
═══════════════════════════════════════════════════
  z_l → φ⁺ → z⁺_l → 解码器 g_ω → ŷ⁺_l  (正变换预测)
  z_l → 直接     → 解码器 g_ω → ŷ_l    (原始预测)
  z_l → φ⁻ → z⁻_l → 解码器 g_ω → ŷ⁻_l  (负变换预测)

═══════════════════════════════════════════════════
第 2 步：计算认知不确定性 U_E（整段信号级别）
═══════════════════════════════════════════════════
  U_E_l = (1/T) Σ KL(ŷ⁺_l, ŷ⁻_l 相对于 ŷ_l)

  如果 U_E_l > α (0.1)  →  ❌ 判定第 l 个导联无效
  如果 U_E_l ≤ α (0.1)  →  ✅ 导联有效，继续分析

═══════════════════════════════════════════════════
第 3 步：多导联融合（如果有多个导联）
═══════════════════════════════════════════════════
  在所有通过第 2 步筛选的导联中，
  选 U_E 最低的那个导联 l* 的预测 ŷ_{l*} 作为最终输出

  直觉：选"最有把握"的那个导联，而不是简单投票

═══════════════════════════════════════════════════
第 4 步：QRS 波检测（逐帧）
═══════════════════════════════════════════════════
  在 ŷ_{l*} 的概率曲线上找局部最大值
  → 得到 B 个 R 峰位置 {R_1, R_2, ..., R_B}

═══════════════════════════════════════════════════
第 5 步：计算偶然不确定性 U_A（逐拍级别）
═══════════════════════════════════════════════════
  对每个 R 峰 R_k：
    取 R_k 前后共 7 帧 (R_k - 3 到 R_k + 4) 的 U_A 平均值

    如果 平均 U_A > β (0.12)  →  ⚠️ 标记为"可疑心跳"
    如果 平均 U_A ≤ β (0.12)  →  ✅ 确定的心跳

═══════════════════════════════════════════════════
最终输出
═══════════════════════════════════════════════════
  1. R 峰位置序列 {R_1, R_2, ..., R_B}
  2. 整段信号的质量标签（ECG / 非ECG）
  3. 每个心跳的可信度标签（确定 / 可疑）
```

## 6. 模型结构细节

### 6.1 编码器 f_θ：多分支 CNN

```
输入 ECG (1D, 2500 采样点 @250Hz)
  │
  ├──→ CNN Branch 1 (小感受野, 捕捉高频细节)
  │
  ├──→ CNN Branch 2 (中感受野, 捕捉中频模式)
  │
  └──→ CNN Branch 3 (大感受野, 捕捉低频趋势)
  │
  ▼
  Merge (合并三个分支)
  │
  ▼
  z ∈ R^{T×d}  (T=625 帧, 每帧 d 维特征)
```

每个分支是带有 SE（Squeeze-and-Excitation）注意力的残差 CNN。三个分支用不同的卷积核大小（类比不同焦距的镜头），分别捕捉不同时间尺度的特征，然后合并。

### 6.2 解码器 g_ω：全连接 + Sigmoid

```
z_τ (单帧特征, d 维)
  │
  ▼
全连接层 × 若干
  │
  ▼
Sigmoid
  │
  ▼
ŷ_τ ∈ [0, 1]  (该帧是 QRS 的概率)
```

解码器非常轻量——对每一帧独立做预测（不考虑帧间关系，这由编码器的卷积来处理）。

### 6.3 正负变换 φ⁺/φ⁻：小型 MLP

```
z_τ (单帧特征, d 维)
  │
  ▼
MLP (几层全连接 + ReLU)
  │
  ▼
z_τ⁺ 或 z_τ⁻ (变换后的特征, 同样 d 维)
```

φ⁺ 和 φ⁻ 结构相同但参数独立。它们不改变特征的维度，只改变特征在空间中的位置。

## 7. 实验结果

### 7.1 噪声筛选（能不能识别"非心电"片段）

| 方法 | ECG 集准确率 | 心音准确率 | PPG 准确率 | 真实噪声准确率 |
|---|---|---|---|---|
| GbSQI | 74.60% | 43.62% | 99.07% | 91.60% |
| kSQI | 83.23% | 1.38% | 54.16% | 51.93% |
| **PN-QRS** | **96.84%** | **97.37%** | **100.00%** | **97.50%** |

PN-QRS 全面碾压传统信号质量评估方法，尤其是对心音和真实噪声的识别。

### 7.2 模糊心跳识别

在人工标注的模糊心跳数据集上，PN-QRS 达到：
- 模拟伪差数据集 (SIM-AM-QRS)：F1 = 61.40%
- 真实伪差数据集 (RW-AM-QRS)：**F1 = 82.41%**

### 7.3 多导联 QRS 检测

| 数据库 | 方法 | FP (误检) | FN (漏检) | F1 |
|---|---|---|---|---|
| MIT-BIH (2导联) | P&T+GbSQI | 5828 | 7965 | 93.64% |
| | MC-Dropout | 137 | 467 | 99.72% |
| | Deep Ensemble | 30 | 196 | 99.90% |
| | **PN-QRS** | **24** | **94** | **99.95%** |
| INCART (12导联) | P&T+GbSQI | 3207 | 3708 | 98.03% |
| | **PN-QRS** | **15** | **9** | **99.99%** |

在 MIT-BIH 上 F1 达到 99.95%，INCART 上仅 15 个误检 + 9 个漏检，几乎完美。

**多导联融合策略**：选择认知不确定性最低的那个导联的结果作为最终输出。这比简单投票更聪明——它挑的是"最有把握"的那个导联。

### 8.2 数据规模对比：PN-QRS vs ECGFounder

| 维度 | PN-QRS（本文） | ECGFounder（我们的项目） |
|---|---|---|
| **训练数据来源** | CPSC2019-Train | MIMIC-IV-ECG |
| **训练数据规模** | ~2000 条 × 10 秒 ≈ **5.6 小时** | ~800K 条 × 10 秒 ≈ **2,222 小时** |
| **训练数据导联数** | 1（单导联） | 12（标准 12 导联） |
| **训练数据采样率** | 500 Hz | 500 Hz（MIMIC-IV 原始） |
| **微调数据** | 无（直接在 QRS 检测上评测） | PTB-XL 等下游任务数据集 |
| **任务** | QRS 波群检测（二分类逐帧） | ECG 基础模型预训练 → 多下游任务微调 |
| **数据量级差距** | — | ECGFounder 训练数据约为 PN-QRS 的 **400 倍** |

### 8.3 关键差异分析

#### 差异 1：任务本质不同

```
PN-QRS:     ECG 信号 → "这一帧是 QRS 还是非 QRS？" (逐帧二分类)
ECGFounder: ECG 信号 → 通用表征 → 心律失常分类 / 年龄预测 / ... (基础模型)
```

PN-QRS 是一个**专用工具**（QRS 检测器），ECGFounder 是一个**通用基础模型**。它们不是竞争关系，而是 PN-QRS 可以作为 ECGFounder 预处理流水线的一个**上游环节**。

#### 差异 2：数据集完全不重叠

| | PN-QRS | ECGFounder |
|---|---|---|
| **预训练/训练** | CPSC2019 (中国竞赛数据) | MIMIC-IV-ECG (美国 ICU 数据) |
| **评测/微调** | MIT-BIH, INCART, QTDB, LUDB | PTB-XL, CPSC2018, Chapman 等 |
| **数据人群** | 中国心律失常患者 (院外) | 美国 ICU 患者 (院内) |

两者**没有任何数据集重叠**。这意味着：
- PN-QRS 的模型不能直接拿来给 ECGFounder 的 MIMIC-IV 数据做 QRS 检测（会有 domain shift）
- 但 PN-QRS 的**方法论**可以在 MIMIC-IV 数据上重新训练

#### 差异 3：数据规模差异巨大

PN-QRS 只用了 ~2000 条 10 秒单导联做训练，总数据量极小。这是因为：
- QRS 检测是相对简单的任务（信号模式明确）
- 单导联处理，不需要学习导联间关系
- 论文重点不在"大数据训练"，而在"不确定性估计方法"

ECGFounder 用 800K 条 12 导联做预训练，数据量大了几个数量级，因为基础模型需要学习 ECG 的通用表征。

#### 差异 4：标注类型不同

| | PN-QRS | ECGFounder |
|---|---|---|
| **标注粒度** | 采样点级（每个 R 峰的精确位置） | 记录级（每条 ECG 的诊断标签） |
| **标注内容** | R 峰坐标（如第 1523 个采样点） | 诊断类别（如"房颤"、"ST 段抬高"） |
| **标注成本** | 中等（需要逐拍标注） | 较低（诊断标签通常已有） |

### 8.5 如果要在我们的流程中复现 PN-QRS

如果我们想把 PN-QRS 的思想应用到 ECGFounder 的预处理流水线中，需要注意：

```
方案 A：直接用 PN-QRS 预训练模型（如果作者开源了权重）
  ├── 优点：零开发成本
  └── 风险：CPSC2019 训练的模型在 MIMIC-IV 数据上可能有 domain shift
             （不同采集设备、不同患者群体、不同导联配置）

方案 B：在 MIMIC-IV 数据上重新训练 PN-QRS
  ├── 前提：MIMIC-IV 需要有 R 峰标注（可用 Pan-Tompkins 等传统方法自动生成伪标签）
  ├── 优点：与我们数据完全匹配
  └── 成本：需要实现 PN-QRS 的训练代码 + ~数小时 GPU 训练

方案 C：仅借鉴思想，不用 PN-QRS 代码
  ├── 用认知不确定性的概念指导数据清洗（但用其他方法实现）
  ├── 例如：用多个传统 QRS 检测器投票，不一致的片段标记为低质量
  └── 优点：最简单；缺点：精度不如 PN-QRS
```

**推荐方案 B**，因为 MIMIC-IV 的数据量足够大，重新训练一个 PN-QRS 模型并不困难，且能确保与下游 ECGFounder 预训练的数据分布完全一致。

## 11. 局限性与未来方向

1. **仅限二分类**：目前只区分 QRS vs 非 QRS，未来希望扩展到多标签分类
2. **伪差识别精度有提升空间**：F1 = 82.41% 说明模糊心跳的识别仍然很难（因为它们本身就难以明确定义）
3. **主动学习潜力**：高认知不确定性、低偶然不确定性的样本最适合加入训练集来提升模型

## 12. 对我们项目（ECGFounder）的启示

### 与预处理流水线的关系

PN-QRS 的核心思想可以直接应用于 ECGFounder 的预训练数据预处理：

1. **数据清洗**：用认知不确定性过滤掉非 ECG 片段（特别是 MIMIC-IV 等大规模数据集中可能存在的无效记录）
2. **质量分级**：不只是二分法（保留/丢弃），而是根据不确定性给数据打分，在训练时用加权采样
3. **伪差标注**：偶然不确定性可以辅助自动标注可疑心跳，减少人工审核工作量

### 技术借鉴

- **类偏置变换的思想**：在 Mamba 的特征空间中引入类似的正负变换，可能有助于模型学习更鲁棒的 ECG 表征
- **不确定性驱动的多导联融合**：12 导联 ECG 中，不同导联的信号质量不同，用不确定性选择最优导联比简单平均更合理

## 13. QRS 检测与端到端分类的关系：为什么我们之前的实验不需要 QRS？

### 13.1 两种任务的根本区别

```
PN-QRS 任务：
  输入: 10秒 ECG 信号
  输出: 每一帧(16ms)是否是QRS波 → 找到每一个心跳的R峰位置
  粒度: 采样点级别（毫秒量级）

ECGFounder / 我们的 Mamba 任务：
  输入: 10秒 ECG 信号（5000 个采样点）
  输出: 这段信号是哪种心梗亚型？(ALMI/AMI/ASMI...)
  粒度: 记录级别（整段信号给一个标签）
```



### 13.2 为什么端到端分类不需要显式 QRS 检测？

因为我们用的是端到端深度学习——模型（Mamba/ECGFounder）直接吃原始 5000 个采样点，自己内部学会了"看哪里"。QRS 波的位置信息已经隐式地编码在卷积/注意力层里，不需要显式地先找出来。

就像 ResNet 做图像分类时，不需要先手动标出"这里是眼睛、这里是鼻子"——网络自己学。

### 13.3 什么时候才需要显式的 QRS 检测？

```
需要 QRS 检测的场景：
  ├── 计算心率、RR间期、HRV  → 必须知道每个R峰的精确时间
  ├── 心律失常（节律分析）    → 需要判断心跳是否规则
  ├── 把信号切成"一拍一拍"   → 逐拍分析
  └── 数据清洗：过滤噪声片段  → PN-QRS 的主要用途

不需要 QRS 检测的场景：
  └── 端到端分类（我们的当前任务）→ 直接喂整段信号给深度模型
```

## 14. 战略分析：QRS 检测对上臂环 MI 检测的必要性

> 背景：我们的最终目标不仅是单导联标准 ECG 的 MI 分类，而是既支持标准 12 导联、又支持上臂环（非标准导联）的心肌缺血检测系统。最终形态是用上臂环进行人的心肌缺血检测。

### 14.1 核心问题：跨域迁移

上臂环场景面临的根本挑战是**从标准导联到非标准导联的知识迁移**：

```
训练时有的数据：                       部署时的目标：
  PTB-XL / MIMIC-IV                     上臂环
  标准 12 导联                           非标准导联
  信号幅度 ~1mV                          信号幅度 ~100-500μV
  高信噪比                               低信噪比（运动伪差多）
  海量标注数据                            几乎没有标注数据
```

**没有任何现成的"上臂环 + MI 标注"数据集。** 不可能直接在上臂环数据上端到端训练，因为没有标签。所以问题变成了：**如何把在标准导联上学到的知识迁移到非标准导联？**

### 14.2 端到端模型在非标准导联上为什么难迁移

端到端模型（Mamba、ECGFounder 等）学到的是**特定导联配置下的波形模式**。比如：

```
Lead I 上的 ST 抬高长这样：            上臂环上的 ST 抬高长这样：
    R                                      r
    /\                                   /\
   /  \_____ ← ST 抬高                  /  \__ ← 也有 ST 变化，但
  /  Q  S                              /       幅度更小、形态不同、
 /                                    /        叠加了更多肌电噪声
```

端到端模型在 Lead I 上学到的特征（卷积核权重）**不能直接用在上臂环信号上**——因为波形形态、幅度、基线都不同。这不是微调几个 epoch 能解决的，是**输入分布的根本性偏移**。

### 14.3 QRS 检测为什么能当"桥梁"

QRS 波群有一个独特的性质：**它是所有导联配置中最鲁棒、最易检测的特征。**

无论是标准 12 导联还是上臂环，QRS 波群都存在且可识别——因为心室除极产生的电场太强了，任何两点的电位差都能捕获到。

一旦检测到了 QRS（R 峰位置），就有了一个**跨导联的时间锚点**，然后可以提取**导联无关或弱导联相关**的特征：

```
导联完全无关的特征（不需要重新训练）：
  ├── R-R 间期序列 → 心率、HRV
  ├── R-R 间期变异性 → 心律是否规则
  └── 心率趋势 → 是否有心动过速/过缓

导联弱相关的特征（少量标注即可校准）：
  ├── QRS 宽度 → 传导阻滞
  ├── QT 间期 → 需要知道 T 波结束点
  └── ST 段相对于基线的偏移方向和幅度 → MI 最核心的特征！

导联强相关的特征（需要大量该导联的标注数据）：
  └── 整体波形模式 → 端到端模型学的就是这个
```

**MI 诊断的核心依据是 ST 段改变。要测量 ST 段，必须先知道 QRS 在哪**——因为 ST 段的定义就是"J 点（QRS 结束点）到 T 波起始之间的区域"。

### 14.4 建议的分阶段技术路线

```
阶段 1（当前）：标准导联 + 端到端
  └── 就是我们正在做的：Mamba + MIMIC-IV 预训练 + PTB-XL 微调
      → 建立 12 导联和单导联 Lead I 的性能基准

阶段 2（近期）：引入显式 QRS 检测作为辅助模块
  ├── 方案 A：多任务学习
  │     主任务：MI 分类（记录级标签）
  │     辅助任务：QRS 检测（帧级标签，可用传统算法自动生成）
  │     → 共享编码器被迫学习心跳结构，表征更鲁棒
  │
  └── 方案 B：两阶段 pipeline
        Stage 1: QRS 检测 → R 峰定位
        Stage 2: 以 R 峰为锚点，提取每拍的 ST 段特征 → 分类
        → 更可解释，更容易跨导联迁移

阶段 3（最终目标）：上臂环部署
  ├── 用标准导联数据训练好 QRS 检测器（迁移到上臂环相对容易）
  ├── 收集少量上臂环数据 + 同步标准 12 导联数据（做配对标注）
  ├── 用 QRS 锚定的特征做跨域校准
  └── 最终：上臂环信号 → QRS检测 → ST段分析 → MI 预警
```

### 14.5 三个关键问题的结论

**Q1：QRS 检测融入端到端分类，能提升标准导联上的性能吗？**

提升可能有限——因为端到端模型已经隐式学会了 QRS 的位置信息。但作为多任务学习的辅助目标，有文献表明可以提升泛化性和对噪声的鲁棒性。更大的价值在于为阶段 3 做铺垫。

**Q2：最终目标是上臂环 MI 检测，是否需要 QRS？**

**几乎是必须的。** 因为没有大规模上臂环标注数据来端到端训练，QRS 检测是从标准导联向非标准导联迁移知识的最自然的桥梁。QRS 是跨导联最不变的特征，以它为锚点提取的 ST 段特征可以在不同导联配置之间迁移。

**Q3：不加 QRS 的端到端在非标准导联上"根本不行"吗？**

不是"根本不行"，但需要大量上臂环 + MI 标注的配对数据才行——而这种数据目前不存在且获取成本极高。QRS 检测让我们可以**用极少的上臂环数据**完成迁移，是实际工程可行性的关键差异。

### 14.6 PN-QRS 在上臂环场景的额外价值

PN-QRS 的不确定性估计在上臂环场景下价值更大：

| 功能 | 标准导联下的价值 | 上臂环下的价值 |
|---|---|---|
| **认知不确定性（噪声筛选）** | 有用但不急迫（信号质量较好） | **极其关键**（运动伪差频繁，需要实时判断"这段信号能不能用"） |
| **偶然不确定性（可疑心跳标记）** | 有用（辅助质控） | **极其关键**（低信噪比下很多心跳形态不明确，需要标记可信度） |
| **多导联融合（选最优导联）** | 有用（12 导联间选最佳） | **直接适用**（如果臂环有多个电极对，选信号质量最好的那对） |

上臂环信号质量远差于标准导联，PN-QRS 的"先判断信号可不可用，再判断心跳可不可信"的两道防线，在这个场景下从"锦上添花"变成了"必需品"。

## 16. 两条技术路线对比分析：3-Loss 多任务 vs QRS 前处理 Pipeline

> 背景：最终目标是上臂环非标准导联的心肌缺血检测。现在面临两条可能的技术路线。

### 16.1 路线 A：3-Loss 多任务学习（Reconstruction + 对比学习 + QRS 检测）

#### 核心思路

在现有 Uni-Mamba 2-Loss 基础上增加第三个 loss——用业界最优算法生成 QRS 伪标签，强迫 Mamba 编码器同时学习：

```
Mamba 编码器 (共享)
  │
  ├──→ 解码器 1: Reconstruction Loss (重建原始信号)
  │
  ├──→ 解码器 2: Contrastive Loss (对比学习，SelfMIS 风格)
  │
  └──→ 解码器 3: QRS Detection Loss (逐帧二分类：是/否 QRS)
                   ↑
                   伪标签来自传统算法（Pan-Tompkins / neurokit2）
```

#### 优势

1. **端到端，不需要额外推理步骤**：部署时只有一个模型
2. **QRS 检测作为归纳偏置**：强迫编码器学习心跳结构，产生的表征天然包含时序定位能力
3. **对非标准导联的潜在迁移优势**：编码器被迫在特征空间中编码 QRS 位置，而 QRS 是跨导联最不变的特征——理论上这种"QRS 意识"可以迁移到上臂环信号
4. **伪标签成本为零**：用传统算法在标准导联上自动生成，无需人工标注
5. **论文故事好讲**："不确定性感知的多任务心电基础模型"

#### 风险与挑战

1. **多任务平衡难度大**：三个 loss 的权重如何设定？QRS 检测是帧级任务（5000 帧），MI 分类是记录级任务（1 个标签），尺度差异巨大
2. **伪标签噪声**：传统算法在 MIMIC-IV 的噪声片段上生成的 QRS 伪标签可能有 2-5% 的错误率
3. **迁移到上臂环不确定**：虽然理论上"QRS 意识"应该能迁移，但没有实验验证
4. **工程复杂度高**：需要修改 Mamba 架构（加分支头），修改训练流程，调三个 loss 权重

#### 数据需求

```
预训练阶段：与现有相同（MIMIC-IV 800K），QRS 伪标签用传统算法自动生成，零额外成本
微调阶段：与现有相同（PTB-XL）
上臂环迁移：需要 50-200 条上臂环 ECG（无标签亦可，用于 domain adaptation）
```

---

### 16.2 路线 B：独立 QRS 检测器 + 清洗后数据喂入 Uni-Mamba

#### 核心思路

两个独立模型，串行 pipeline：

```
Stage 1: QRS 检测器（独立模型）
  输入: 原始 ECG 信号
  输出:
    ├── R 峰位置序列
    ├── 信号质量评分（认知不确定性）
    └── 每拍可信度（偶然不确定性）

Stage 2: Uni-Mamba 2-Loss（现有模型，不修改）
  输入: Stage 1 筛选和清洗后的干净 ECG
  输出: MI 亚型分类
```

#### 优势

1. **模块化，风险隔离**：两个模型独立，Stage 1 出问题不影响 Stage 2
2. **各自优化到极致**：不存在多任务平衡问题
3. **数据清洗的直接价值**：过滤噪声片段后，训练数据质量提升，可能直接提升现有 AUROC
4. **上臂环迁移更可控**：QRS 检测器可以单独用上臂环数据微调（任务简单，少量数据即可）
5. **工程简单**：不需要修改 Mamba 架构，只加一个前处理步骤

#### 风险与挑战

1. **两阶段误差累积**：Stage 1 的错误直接影响 Stage 2 的输入
2. **推理延迟翻倍**：部署时需要跑两个模型
3. **信息损失**：过滤或裁剪信号时可能丢弃有用信息

#### 数据需求

```
QRS 检测器训练：CPSC2019 ~2000 条（或 MIMIC-IV 上用传统算法生成伪标签）
MI 分类：与现有相同
上臂环迁移：需要 100-500 条上臂环 ECG，其中至少 20-50 条有配对 12 导联
```

### 16.3 两条路线的直接对比

| 维度 | 路线 A（3-Loss 多任务） | 路线 B（独立 QRS + Pipeline） |
|---|---|---|
| **对标准导联性能的影响** | 可能小幅提升（QRS 辅助任务正则化） | 可能提升（数据清洗改善训练集质量） |
| **工程改造量** | **大**（改架构、加分支头、调三权重） | **小**（加一个前处理步骤） |
| **部署复杂度** | 低（一个模型） | 中（两个模型串行） |
| **上臂环迁移难度** | **不确定**（理论上有优势，但未验证） | **更可控**（QRS 检测器可单独微调） |
| **论文价值** | **高**（多任务+不确定性，创新点多） | 中（工程贡献为主） |
| **标准导联数据需求** | 与现有相同（零额外成本） | 与现有相同 + QRS 标注（可自动生成） |
| **上臂环数据需求** | **较少**（50-200 条） | **较多**（100-500 条，需部分配对 12 导联） |
| **多任务平衡风险** | **高**（三个 loss 尺度不同） | 无（两个模型独立） |
| **调试难度** | **高**（不确定是哪个 loss 出了问题） | 低（Stage 1 和 2 独立检查） |
| **可行性（快速验证）** | 中（需 2-4 周改架构+实验） | **高**（1-2 周可看到初步结果） |

### 16.4 关于数据需求的深入分析

**路线 A（3-Loss）的上臂环数据需求确实更少。** 原因：

```
路线 A 的迁移逻辑：
  标准导联上：编码器同时学了 MI 分类 + QRS 检测
  → 编码器的特征空间中已经编码了心跳结构
  → 换到上臂环时，只要信号中有可识别的 QRS
  → 编码器可能"自动"找到锚点，无需大量新数据
  → 但这是理论推测，需要实验验证

路线 B 的迁移逻辑：
  Step 1: 用上臂环数据微调 QRS 检测器（需要 R 峰标注，至少 50-100 条）
  Step 2: 用微调后的 QRS 检测器提取 ST 段特征
  Step 3: 用配对的标准导联数据校准 ST 段特征的映射关系
  Step 4: 用少量有 MI 标签的上臂环数据微调分类器
  → 每一步都需要一些数据，累积起来需求量更大
```

但路线 B 的**每一步都更可控、可验证**——如果某步失败，你确切知道是哪里的问题。

### 16.5 推荐策略：先 B 后 A，两条腿走路

```
短期（1-2 周）：走路线 B
  ├── 复现/训练一个 QRS 检测器（PN-QRS 或 Yun 2022）
  ├── 用它清洗 MIMIC-IV 预训练数据
  ├── 看清洗后的数据是否提升 Uni-Mamba 的 AUROC
  └── 这一步无论如何都有价值（数据清洗改善基线）

中期（2-4 周）：走路线 A
  ├── 在 Mamba 上加 QRS 检测头（第三个 loss）
  ├── 在标准导联上对比 2-Loss vs 3-Loss 的性能
  └── 如果 3-Loss 有提升 → 发论文的素材

长期（有上臂环数据后）：两条路线都试
  ├── 路线 B：用微调的 QRS 检测器处理上臂环信号 → 喂入 Mamba
  ├── 路线 A：直接把上臂环信号喂入 3-Loss Mamba → 看能否 zero-shot 迁移
  └── 哪条路效果好用哪条（大概率两条互补）
```

**先做路线 B 的原因**：
1. 风险低，1-2 周可出结果
2. 数据清洗本身就能提升基线
3. QRS 检测器是后续所有工作的基础设施
4. 即使路线 A 最终更好，路线 B 的 QRS 检测器仍然有用（生成伪标签给路线 A 用）

### 17.7 复现 vs 论文差距的根因分析

**结论：差距不是因为训练数据不同（论文也用 CPSC2019-Train 2000 条），而是三个具体的评测/实现差异。**

#### 差异 1（最关键）：多导联融合 vs 单导联

| | 论文 | 我们的复现 |
|---|---|---|
| **导联数** | **2 导联**（MLII + V5），选不确定性最低的导联 | **1 导联**（仅 MLII） |
| **Table III 标注** | "2 Leads" | 单导联 |

论文 Table III 明确标注 MIT-BIH 评测使用 "**2 Leads**"，且 Algorithm 1 说明："choose the output from the optimal lead with the **minimum uncertainty** frame-by-frame"。这意味着对每一帧，论文选择两个导联中不确定性更低的那个预测结果。

**影响**：当 MLII 导联上某个心跳形态异常（如 Record 207 的室性早搏）时，V5 导联可能仍然清晰可检。多导联融合直接减少了 FN。

**这是最大的差距来源。** 我们的评测仅用了 1 个导联，相当于放弃了论文 50% 的信息量。

#### 差异 2（重要）：窗口重叠 vs 非重叠

| | 论文 | 我们的复现 |
|---|---|---|
| **窗口策略** | 2 秒重叠（论文 Section IV-C-2 明确说明） | 非重叠（10 秒不重叠切分） |

论文原文（第 8 页）："a **2-second overlap** is adopted for long-term ECGs"。

**影响**：非重叠切分时，恰好落在窗口边界的心跳会被切断导致漏检。估算：
- 44 条 × 30 分钟 = 约 7,920 个 10 秒窗口
- 每个窗口有 2 个边界，每个边界附近约有 1 个心跳
- 约 5-10% 的边界心跳会被漏检 → 估计贡献 ~400-800 FN

2 秒重叠可以消除大部分边界效应（重叠区域的心跳会在相邻两个窗口中都被检测到，取其一即可）。

#### 差异 3（次要）：训练 batch size

| | 论文 | 我们的复现 |
|---|---|---|
| **Batch size** | 400（论文默认配置） | 32（因 GPU 显存被其他任务占用） |

**影响**：PN-QRS 用 Monte-Carlo 近似估计类别边际分布 r_p 和 r_n（Eq. 10）：`r ≈ (1/N) Σ z^i ⊗ MASK`。Batch size 越大，近似越准确。batch=32 时每个 batch 中可能只有 3-5 个 QRS 帧，r_p 的估计方差很大，导致正负变换的训练质量下降。

#### 排除的因素

| 可能的因素 | 实际情况 | 是否是原因 |
|---|---|---|
| **训练数据量不同** | 论文也用 CPSC2019-Train 2000 条 | **否** |
| **更精细的超参** | 论文用 BeatConfig 默认超参 | **可能微小影响，不是主因** |
| **预处理不同** | 论文 250Hz → 625 帧，复现 200Hz → 500 帧（preprocess_ecg 内部逻辑） | **可能有影响，但不是主因** |
| **不确定性后处理** | 已接入 `uncertain_est()` + `correct()` | **否**（已使用） |

### 17.8 弥补差距的行动方案与实测对比

| 行动 | 事前预测 F1 提升 | **实测 F1 提升** | 预测偏差 |
|---|---|---|---|
| **加入 V5 导联 + 不确定性导联选择** | +1.0-1.5pp（最大提升） | **+0.0pp**（Se +0.03pp，FP +47） | ⚠️ 大幅高估 |
| **改为 2 秒重叠窗口** | +0.3-0.5pp | **+1.01pp** | ⚠️ 大幅低估 |
| **增大训练 batch size / 真实数据** | +0.1-0.3pp | **≈0pp** | 高估 |
| **累计** | 预期 0.995+ | **实测 0.9969** | 总量级接近 |

> **教训**：事前归因分析把主因判给了双导联（60%），实测最大贡献项是窗口重叠。这说明在没有对照实验时，按直觉排优先级可能完全反转。详见 17.12 节的实证分析。

### 17.10 评测改进实验（2026-03-28 更新）

#### 改进 v2：2 秒重叠窗口

#### 改进 v3：R 峰信号精化

**脚本**：`evaluate_mitbih_v3.py`

**原理**：模型在 50Hz 上预测，上采样到 360Hz 时精度为 ±7.2 samples。对每个检测到的 R 峰位置，在原始 360Hz 信号中 ±25ms（±9 samples）范围内搜索 |signal| 最大值作为精化位置。

**结果**：v3 Se=P+=F1 与 v2 **完全相同**（F1=99.69%）。

**原因**：上采样引入的位置误差（±7 samples ≈ 19ms）已在 150ms tolerance 范围内，精化不影响 TP/FP/FN 计数。若使用更严格的 tolerance（如 75ms 或 50ms），v3 可能显现优势。

| 数据集 | 记录数 | Se | P+ | F1 | 备注 |
|--------|--------|-----|-----|-----|------|
| CPSC2019-Test | 200 | **0.9992** | 0.9090 | 0.9520 | P+ 偏低：同一 10s 段中 FP 被放大 |

| LUDB | 200 | **0.9984** | 0.8296 | 0.9062 | P+ 偏低，10s 慢心率记录 |

#### LUDB P+ 偏低的机制解释

LUDB 每条记录仅 10 秒，心率偏慢（约 6-15 次/窗口）。模型在窗口首尾各会产生 1-2 个边界误检（FP），在只有 6 个 R 峰的记录中 FP/ref 比例被严重放大：

```
# 正常长时程记录（MIT-BIH）
每个 10s 窗口 ≈ 12 beats；1 FP 占比 = 8%

# LUDB 短记录
每条记录 ≈ 8 beats；1 FP 占比 = 12.5%
→ 200 条累积导致 P+ 从 ~99% 降至 82.96%
```

**结论**：LUDB 的低 P+ 是**短记录边界效应**，不代表模型真实精度，使用 2s 重叠窗口或去除边界预测可改善。

论文 Table I 报告了 PN-QRS 在四类信号上的噪声筛选准确率（Acc%）：

| 数据集类型 | PN-QRS（论文） | 说明 |

#### 剩余 0.26pp 差距的真实原因

```
剩余 FN = 464
  Record 207 贡献 FN = 234（50.4%）

Record 207 特征：
  - 左束支传导阻滞（LBBB）
  - 宽 QRS（>120ms），呈扭曲形态
  - 2:1 房室传导阻滞（部分 P 波无对应 QRS）

```

**要达到论文 F1=99.95%，最可能的路径**：
- 论文可能在 MIT-BIH 训练数据中包含了少量 LBBB 样本，或使用 LBBB 数据增强
- 或论文在报告 MIT-BIH 结果时使用了 leave-one-out 或 train-on-MIT-BIH 的设置（与 CPSC2019 不同的训练集）

#### 工程实现层面总结

**可控的工程差异已全部消除**：
- ✅ 2s 重叠窗口（+1.01pp F1）

### 18.2 PN-QRS 可以做什么？（三种用法）

```
用法 A: 噪声过滤器（数据清洗）
─────────────────────────
800K ECG → PN-QRS 噪声筛选 → 去掉噪声片段 → 更干净的预训练数据
                                    ↓
                         预期: AUROC +0.1-0.5pp

用法 B: R-peak 定位器（信号分割）
─────────────────────────
上臂环 ECG → PN-QRS R-peak → 切割为逐拍信号 → 逐拍分类（MI/正常）
                                    ↓
                         预期: 非标准导联下更鲁棒

用法 C: 不确定性信号质量评分（SQI）
─────────────────────────
上臂环 ECG → PN-QRS 不确定性 → 逐帧 SQI 分数 → 自动舍弃低质量段
                                    ↓
                         预期: 可穿戴场景下减少误报
```

#### 用法 A：数据清洗——最短路径，最低风险

**原理**：MIMIC-IV-ECG 800K 条中不可避免地混入噪声片段（导联脱落、基线漂移、肌电干扰）。PN-QRS 的认知不确定性 U_E 可以识别"模型从未见过的模式"——如果一个 10s 窗口的平均 U_E 远高于正常 ECG，极可能是噪声。

**操作**：
```python
# 对 800K 数据逐条运行 PN-QRS
for record in mimic_800k:
    logits = model(record)
    uc = uncertain_est(logits)
    mean_uc = uc.mean()
    if mean_uc > threshold:
        mark_as_noisy(record)
# 阈值可通过 CPSC2019-Test 的干净数据统计得到
```

**收益估计**：
- 论文 Table I 噪声筛选准确率 96.84%（ECG 集）、100%（PPG）、97.50%（真实噪声）
- 若 800K 中 5% 是噪声片段，清洗后约去掉 ~40K 条
- **对预训练质量的提升取决于噪声比例**：若噪声严重干扰了对比学习的负样本采样，清洗后 AUROC 可能提升 0.1-0.5pp
- 工作量：**约半天**（写一个 batch 推理脚本 + 统计分析）

**风险**：
- 如果 MIMIC-IV-ECG 本身质量已较高（医院级采集），噪声比例可能仅 1-2%，清洗收益微小
- 需要先在一个小样本上验证噪声比例

#### 用法 B：R-peak 分割——上臂环的关键价值

**原理**：上臂环非标准导联的 QRS 形态与标准 12 导联不同（振幅低、T 波可能与 QRS 等高）。端到端模型在固定窗口上分类时，如果窗口没有对齐心拍中心，分类性能会下降。PN-QRS 的 R-peak 检测可以让窗口精确对齐心拍。

**接入方式**：
```
上臂环 ECG（连续采集）
    ↓
PN-QRS R-peak 检测（已验证在多采样率下泛化良好）
    ↓
以 R-peak 为中心切出固定长度窗口（如 ±350ms）
    ↓
送入 Uni-Mamba 分类器
    ↓
逐拍 MI 概率 → 30 秒/1 分钟滑窗投票 → 最终告警
```

**关键优势**：PN-QRS 在 250Hz（QTDB）、360Hz（MIT-BIH）、500Hz（CPSC2019/LUDB）上均表现良好，**采样率泛化已验证**。上臂环即使使用非常规采样率，PN-QRS 也大概率可以直接迁移。

**风险**：
- 上臂环 ECG 信噪比远低于临床采集，PN-QRS 是否还能保持 >95% F1 需要实测
- 如果 R-peak 形态过于异常（极低振幅），可能需要在上臂环数据上微调

#### 用法 C：信号质量评分（SQI）——降低误报的安全网

**原理**：逐帧认知不确定性可以作为信号质量指标。在运动干扰、皮肤接触不良等场景下，不确定性会飙升——系统自动暂停分析而非输出不可靠结果。

**在 MI 检测产品中的价值**：
- 漏报（FN）危害大于误报（FP）——但频繁误报导致用户忽视告警
- SQI 可以在信号质量差时**主动提示"当前信号不可分析"**，而非给出一个低置信度的判断
- 这是医疗器械审批中关注的关键能力

## 19. 论文的完整能力回溯：PN-QRS 不只是 QRS 检测器

> 前面的复现工作（Section 17）聚焦于 QRS 检测性能（MIT-BIH F1=99.69%），这只是论文的"层次 3"。回溯论文原文发现，**PN-QRS 真正的核心贡献是一个三层级的 ECG 信号质量管理系统**，而 QRS 检测仅是其中最后一环。

### 19.1 论文实际做的三件事

```
层次 1: 噪声筛选（Noise Screening）
━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  24小时 Holter 连续信号
       ↓ 切成 10s 片段
       ↓ 计算认知不确定性 U_E（Eq.5）
       ↓ U_E > α (=0.1) → 判为"非 ECG 片段"（丢弃）

  论文结果（Table I）:
  ┌────────────────┬────────┬────────┬────────┐
  │ 方法           │ ECG 集 │ PPG    │ 真实噪声│
  ├────────────────┼────────┼────────┼────────┤
  │ GbSQI          │ 74.60% │ 99.07% │ 91.60% │
  │ kSQI           │ 83.23% │ 54.16% │ 51.93% │
  │ PN-QRS         │ 96.84% │ 100.0% │ 97.50% │
  └────────────────┴────────┴────────┴────────┘

  核心优势: 比 GbSQI 高 6%，比 kSQI 高 13%
  原理: U_E 高 = "模型从未见过这种模式" = 不是正常 ECG

层次 2: 模糊心跳识别（Ambiguous Beat Detection）
━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  通过层次 1 的"看起来像 ECG"的片段
       ↓ 检测 R 峰位置
       ↓ 对每个 R 峰计算偶然不确定性 U_A（Eq.6）
       ↓ U_A > β (=0.12) → 标记为"模糊心跳"（可能是伪迹插入）

  论文结果（Table II）:
  ┌────────────┬────────────────────┬────────────────────┐
  │ 数据集     │ 确定心跳 F1        │ 模糊心跳 F1        │
  ├────────────┼────────────────────┼────────────────────┤
  │ SIM-AM-QRS │ 99.13%             │ 61.40%             │
  │ RW-AM-QRS  │ 99.38%             │ 82.41%             │
  └────────────┴────────────────────┴────────────────────┘

  原理: U_A 高 = "数据本身就模糊" = QRS 形态与伪迹难以区分

层次 3: 多导联 QRS 精确定位
━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  通过层次 1+2 的干净心跳
       ↓ 多导联逐帧选最低不确定性（Algorithm 1）
       ↓ 精确 R 峰位置
  结果: MIT-BIH F1=99.95%, INCART F1=99.99%
```

### 19.2 论文的数据来源：真实 24 小时 Holter

论文 Section IV-A（第 6 页）明确写道：

> "With written consent previously approved by the Bioethics Committee of the First Affiliated Hospital of Nanjing Medical University (2014-SR-090), we have recorded the **24-hour out-of-hospital ECG signals of 200 patients** with a history of arrhythmia disease using a **wearable ECG monitoring device**. From these recordings, **3000 invalid non-ECG episodes** with sporadic QRS-complexes were selected for evaluation."

这意味着：
- 论文的噪声筛选实验**不是在实验室数据上做的**，而是从 200 名心律失常患者的**真实 24 小时 Holter 记录**中提取的
- 3000 段无效片段是从连续记录中挑出来的（含肌电干扰、电极脱落、运动伪迹等）
- 附加测试了带内高斯噪声、心音、PPG、真实环境噪声四类非 ECG 信号

### 19.4 为什么需要 Holter 数据来完整复现

PN-QRS 的噪声筛选能力（Table I 的 96.84%）是在以下条件下测量的：
- **正样本**：CPSC2019-TEST 的干净 ECG（3132 段）
- **负样本**：24 小时 Holter 中筛选出的 3000 段无效片段 + 高斯噪声 + 心音 + PPG

我们目前只有 CPSC2019 数据（干净 ECG），**缺少"脏数据"来测试筛选能力**。要验证层次 1，必须有真实的噪声 ECG 片段。

**获取噪声数据的途径**：

| 数据源 | 导联 | 时长 | 噪声特点 | 下载 |
|--------|------|------|---------|------|
| **INCART**（PhysioNet） | **12 导联** | 30min × 75 条 | Holter 采集，含运动伪迹 | `wfdb.dl_database('incartdb')` |

## 20. 评测数据集详解与 PN-QRS 在 SOTA 中的定位

### 20.1 INCART 和 NSTDB 的标注情况

#### INCART（St. Petersburg 12 导联心律失常库）

| 属性 | 详情 |
|------|------|
| 来源 | St. Petersburg Institute of Cardiological Technics |
| 记录数 | 75 条（来自 32 名 Holter 记录） |
| 导联 | **12 标准导联** |
| 时长 | 30 分钟/条 |
| 采样率 | 257 Hz |
| 标注 | **175,000+ 个心跳标注**（自动检测 + 人工校正） |
| 患者 | 17 男 + 15 女，18-80 岁，冠心病检查患者 |

**标注内容**：有完整的 R 峰位置标注（`.atr` 文件），可直接用于 QRS 检测评测。

**没有的标注**：没有逐段的"噪声/干净"标签。作为 Holter 采集数据，部分片段天然含有运动伪迹和基线漂移，但没有人工标注哪些窗口是噪声。

**用途**：
- QRS 检测评测：直接可用（论文 Table III 用了它，**F1=99.99%**）
- 噪声筛选评测：不能直接用——需要间接方法（见 20.2 节）

#### NSTDB（MIT-BIH 噪声压力测试库）

| 属性 | 详情 |
|------|------|
| 来源 | MIT-BIH / PhysioNet |
| 设计思想 | 把**已知干净 ECG** 叠加**标定强度的噪声** |
| 噪声类型 | 基线漂移（bw）、肌电干扰（ma）、电极运动（em） |
| SNR 级别 | 24dB、18dB、12dB、6dB、0dB、-6dB |
| 原始干净信号 | 118e00、119e00（MIT-BIH 记录 118、119 的干净版） |

**数据结构**：

```
NSTDB/
├── 118e00.dat   ← 干净 ECG（基准）
├── 118e06.dat   ← 干净 ECG + 6dB 噪声
├── 118e12.dat   ← 干净 ECG + 12dB 噪声
├── 118e18.dat   ← 干净 ECG + 18dB 噪声
├── 118e24.dat   ← 干净 ECG + 24dB 噪声
├── 118e_6.dat   ← 干净 ECG + -6dB 噪声（噪声比信号还强）
├── bw.dat       ← 纯基线漂移噪声
├── ma.dat       ← 纯肌电噪声
└── em.dat       ← 纯电极运动噪声
```

**关键优势**：噪声类型和强度**完全已知**。可以精确测量"在 SNR=6dB 时，PN-QRS 还能检出多少 R 峰？U_E 是否正确飙升？"

### 20.2 噪声筛选评测怎么做（不需要逐段噪声标注）

论文 Table I 的评测设计很巧妙——**不需要对 Holter 数据做噪声标注**，而是用"已知身份"的数据做二分类：

```
正样本池（已知是干净 ECG）           负样本池（已知不是正常 ECG）
┌──────────────────────────┐     ┌──────────────────────────┐
│ CPSC2019-Test            │     │ 24hr Holter 中挑出的     │
│ 3132 段干净 10s ECG      │     │ 3000 段无效片段          │
│                          │     │ + 带内高斯噪声           │
│                          │     │ + 心音信号（PCG）        │
│                          │     │ + PPG 光电脉搏波         │
└──────────────────────────┘     └──────────────────────────┘
          ↓                                ↓
     U_E 应该 < α (=0.1)           U_E 应该 > α (=0.1)
          ↓                                ↓
       统计: 正确分类了多少？→ 准确率
```

**我们可以复刻这个设计**：
- **正样本**：CPSC2019-Test 干净 ECG（已有 ✅）
- **负样本**：NSTDB 的纯噪声信号（bw, ma, em）+ NSTDB 低 SNR 片段（0dB, -6dB）

这样完全不依赖 INCART 的噪声标注。INCART 的价值在于：作为"真实 Holter 数据"，可以观察 U_E 的自然分布，验证阈值 α=0.1 在实际场景下的行为。

### 20.3 PN-QRS 是不是当前最好的？

#### QRS 检测精度：第一梯队，但不是唯一

MIT-BIH 上的 QRS 检测 F1 已接近饱和，多个方法在 99.8-99.99% 之间：

| 方法 | MIT-BIH F1 | 年份 | 类型 |
|------|-----------|------|------|
| Pan-Tompkins | ~99.5% | 1985 | 传统（微分+阈值） |
| Hamilton-Tompkins | ~99.7% | 2002 | 改进传统 |
| U-Net + BiLSTM | 99.83% | 2020 | 纯深度学习 |
| Deep Ensemble (×5) | 99.90% | 2021 | 集成不确定性 |
| **PN-QRS** | **99.95%** | 2024 | 不确定性感知（单模型） |
| 多个方法在 INCART 上 | 99.99% | — | 12 导联上饱和 |

**99.95% 是顶尖水平**，但 MIT-BIH 48 条记录只有约 10 万个心跳，在 99.9%+ 区间的差异可能只是几十个心跳的差别，统计显著性存疑。

#### PN-QRS 真正独特的能力：噪声筛选 + QRS 检测一体化

```
                        QRS 检测    噪声筛选     模糊心跳识别   推理开销
                       (Table III)  (Table I)   (Table II)
────────────────────────────────────────────────────────────────────────
Pan-Tompkins            ~99.5%      ❌ 无此能力  ❌ 无此能力    极低
U-Net + BiLSTM          99.83%      ❌ 无此能力  ❌ 无此能力    低
MC-Dropout              99.72%      ⚠️ 需多次推理 ⚠️ 需多次推理  高（×N次）
Deep Ensemble (×5)      99.90%      ⚠️ 需5个模型 ⚠️ 需5个模型  高（×5）
GbSQI（传统 SQI）        N/A        74.60%       ❌ 无此能力    低
kSQI（传统 SQI）         N/A        83.23%       ❌ 无此能力    低
PN-QRS                  99.95%      96.84%       82.41%        低（单次）
```

**关键对比**：

1. **传统 QRS 检测器**（Pan-Tompkins 等）：只能回答"R 峰在哪"，不能判断信号是否是 ECG
2. **传统 SQI 方法**（GbSQI, kSQI）：只能判断信号质量，不能检测 R 峰
3. **MC-Dropout / Deep Ensemble**：理论上能做不确定性估计，但需要多次推理（×N 或 ×5），计算量是 PN-QRS 的 5-50 倍
4. **PN-QRS**：**一个模型、一次推理**，同时输出三个结果——信号是否是 ECG（U_E）、R 峰位置、每个心跳是否模糊（U_A）

#### 噪声筛选能力对比（论文 Table I 原始数据）

| 方法 | ECG 集识别率 | 心音（PCG）| PPG | 真实噪声 |
|------|------------|-----------|------|---------|
| GbSQI | 74.60% | 43.62% | 99.07% | 91.60% |
| kSQI | 83.23% | 1.38% | 54.16% | 51.93% |
| **PN-QRS** | **96.84%** | **97.37%** | **100.00%** | **97.50%** |

- kSQI 对心音（1.38%）和 PPG（54.16%）几乎完全失效——因为这些信号也有周期性，骗过了基于波形分布的判断
- GbSQI 对 ECG 集本身只有 74.60%——把 25% 的正常 ECG 误判为噪声
- PN-QRS 全面领先 13-53 个百分点

### 20.4 "接近完美"意味着什么？

**QRS 检测**：在标准数据集上确实接近完美（F1 > 99.9%）。但要注意：
- 标准数据集（MIT-BIH）是 1980 年代在医院环境采集的，信噪比远高于现代可穿戴设备
- **在真实 Holter 或可穿戴场景下，任何方法的性能都会显著下降**
- NSTDB 在 SNR=0dB 时，传统方法 F1 可以降到 <90%

**噪声筛选**：96.84% 很高但不完美——仍有 3.16% 的正常 ECG 被误判为噪声。在大规模数据清洗中，这意味着每 10 万条会误丢 3160 条有效数据。

**实际场景中的期望性能**：

```
场景                    QRS 检测 F1 预期    噪声筛选准确率预期
────────────────────────────────────────────────────────
医院临床 12 导联         99.9%+              N/A（不需要筛选）
Holter 24小时（安静段）  99.5-99.9%          95%+
Holter 24小时（运动段）  95-99%              85-95%
上臂环可穿戴             90-98%（待验证）    80-90%（待验证）
```

**结论**：PN-QRS 在标准基准上"接近完美"，但在真实可穿戴/Holter 场景下的性能是待验证的——这正是我们要用 INCART 和 NSTDB 做实验的原因。

### 21.2 为什么多导联更好？

多导联 QRS 检测的优势来自三个层面：

**层面 1：信息冗余**
```
同一个心跳在不同导联上的 QRS 形态：

Lead II:    ___/\___    ← R 峰清晰
Lead V1:    ___/\/\___  ← R 峰呈 rSR' 形态，容易漏检
Lead V5:    ___/\___    ← R 峰清晰

→ 如果 V1 漏检了，Lead II 或 V5 仍能检出
→ 多导联融合 = 多次独立机会
```

**层面 2：导联选择（PN-QRS Algorithm 1）**
- 逐帧比较各导联的不确定性
- 选择 U_E 最低的导联作为该帧的预测结果
- **效果**：哪个导联对这个心跳最有信心，就用哪个

**层面 3：异常形态互补**
- LBBB（左束支传导阻滞）在 V1 导联呈 QS 型，可能被误判
- 但在 V5/V6 导联仍呈宽大 R 波，可被正确检出
- 单导联无法获得这种互补信息

### 21.3 非标准导联的现有测试集

**结论：目前没有统一的非标准导联 QRS 检测基准，但有零散的验证数据。**

| 设备类型 | 导联配置 | 已有数据集/验证 | QRS 检测性能 | 来源 |
|---------|---------|---------------|-------------|------|
| **Apple Watch** | 单导联（胸-手） | 临床验证研究 | QRS 间期误差 18.9±16.3ms | JACC 等多篇 |
| **贴片式（Zio Patch 等）** | 单导联（双极） | 99.8% Se vs Lead II | 干净信号时接近标准导联 | Springer 2022 |
| **远程医疗手持设备** | 单导联（手-手） | SAFER-ECG（479 条） | F1=0.98（高质量）/ **0.78-0.84（低质量）** | PLOS Digital Health |
| **6 导联可穿戴** | 6 导联 | AF 筛查研究 | 显著优于单导联（减少假阴性） | Frontiers 2023 |
| **双极臂环** | 非标准单导联 | **无公开测试集** | **未知（待验证）** | — |

#### 关键数据：信号质量对性能的影响

SAFER-ECG 研究（2024，PLOS Digital Health）给出了最直接的证据：

```
信号质量等级          QRS 检测 F1
────────────────────────────────
高质量（干净）          0.98
中等质量（轻度噪声）    0.90-0.95
低质量（运动/接触差）   0.78-0.84
极低质量（接近纯噪声）  < 0.70

→ 从高质量到低质量，F1 下降约 20 个百分点！
```

**这个 20pp 的落差比导联数量（0.66pp）的影响大得多。** 对于可穿戴场景，**信号质量才是决定性能的主要因素**，而非导联配置。

### 21.4 对上臂环场景的推论

上臂环的挑战是**双重劣势叠加**：既是非标准导联（形态失真），又是低信号质量（运动伪迹、皮肤接触不良）。

```
                    标准 12 导联    标准 2 导联    单导联贴片    上臂环（预估）
                    （医院）        （Holter）     （高质量）
──────────────────────────────────────────────────────────────────────
信号质量             极高           高-中          高-低         中-低
导联标准化程度        标准           标准           接近标准      非标准
QRS 形态可识别度      极高           高             高-中         中-低
────────────────────────────────────────────────────────────────────
预期 QRS F1          99.9%+         99.5-99.9%     95-99%       **85-95%**
预期噪声筛选需求      不需要         偶尔需要       经常需要      **几乎必须**
```

**上臂环预估 QRS F1 = 85-95%**，基于两个参考点：
- 单导联高质量已经是 ~98%（贴片）
- 低质量信号再降 10-20pp（SAFER-ECG 证据）
- 非标准导联额外降 1-3pp（形态失真）

### 21.5 这意味着什么：PN-QRS 的噪声筛选能力是刚需

```
传统 QRS 检测器（如 Pan-Tompkins）在上臂环上：
  ┌────────────────────────────────────────┐
  │ 输入: 上臂环 ECG（噪声多、形态异常）   │
  │                ↓                       │
  │ Pan-Tompkins: "这是 R 峰" / "这不是"   │
  │                ↓                       │
  │ 问题: 在噪声段，大量误检和漏检          │
  │ 结果: F1 ≈ 75-85%                     │
  └────────────────────────────────────────┘

PN-QRS 在上臂环上：
  ┌────────────────────────────────────────┐
  │ 输入: 上臂环 ECG（噪声多、形态异常）   │
  │                ↓                       │
  │ U_E > 0.1? → "这段信号不可靠，跳过"    │  ← 层次 1: 先丢弃噪声段
  │                ↓（只处理干净段）         │
  │ U_A > 0.12? → "这个心跳形态模糊，标记" │  ← 层次 2: 标记不确定心跳
  │                ↓（只信任干净心跳）       │
  │ R 峰定位                               │  ← 层次 3: 精确检测
  │                ↓                       │
  │ 在"有效段"上的 F1 可能仍有 95%+         │
  │ 虽然丢弃了部分数据，但保留的数据质量高    │
  └────────────────────────────────────────┘
```

**核心逻辑**：在信号质量不可控的场景下，**"知道什么时候不该检测"比"勉强检测所有数据"更有价值**。这恰恰是 PN-QRS 的不确定性估计提供的——其他 QRS 检测器做不到。

### 21.6 小结

| 问题 | 回答 |
|------|------|
| 99.9%+ 都是多导联吗？ | **是的**，MIT-BIH（2L）、INCART（12L）上达到 99.95-99.99%；单导联 CPSC2019 最高 99.29% |
| 单导联 vs 多导联差多少？ | F1 差 0.66-0.70pp（导联数本身），但**信号质量差导致 F1 下降 10-20pp**，远大于导联数影响 |
| 非标准导联有测试集吗？ | **没有统一基准**，零散验证：Apple Watch 18.9ms 误差，贴片 99.8% Se，SAFER-ECG（F1=0.78-0.98） |
| 上臂环预期性能？ | **QRS F1 ≈ 85-95%**（非标准 + 中低信号质量），噪声筛选是刚需 |
| PN-QRS 的价值？ | **不是 QRS 精度**（差别不大），而是**噪声筛选能力**——在不可控信号质量下，知道何时不该检测 |

## 22. 本地噪声筛选实验结果（2026-03-29）

### 22.1 实验设计

使用已训练的 PN-QRS 模型（`experiments/logs_real/zy2lki18/models/best_model.pt`，CPSC2019 真实数据训练），在三类数据集上评测噪声筛选能力：

| 数据集 | 类别 | 样本数 |
|--------|------|--------|
| CPSC2019-Test | 干净 ECG（单导联，中国 Holter） | 1800 条记录 |
| NSTDB em/ma/bw | 纯合成噪声（肌电/运动伪影/基线漂移） | 各 15 条 × 3 = 45 条 |
| INCART | 12导联 30min Holter（部分噪声段） | 36 条记录（下载 109/225 文件时测试） |

**方法**：对每个 10 秒窗口，计算所有帧的平均 epistemic uncertainty U_E（`entropy(softmax(logits))`），用 CPSC2019-Test 的 P90 作为阈值。

### 22.2 阈值标定

```
CPSC2019-Test 干净 ECG：
  mean U_E = 1.3877
  std U_E  = 0.5016
  P90 阈值 = 1.9967   ← 用作噪声/干净分界线
```

**注意**：论文的 α=0.1 对应概率空间 [0,1]；我们的模型输出 raw logits 经 entropy 计算，U_E 范围约 1.0-5.0，不能直接套用 0.1 阈值。P90 标定是适配我们模型输出尺度的折中方案。

### 22.3 NSTDB 纯噪声结果（Se = 噪声召回率）

| 噪声类型 | 平均 U_E | 被标记为噪声数 | 共计 |
|----------|---------|--------------|------|
| em（肌电） | 3.116 | 15 | 15 |
| ma（运动伪影） | 3.887 | 15 | 15 |
| bw（基线漂移） | 4.219 | 15 | 15 |
| **合计** | **3.741** | **45** | **45** |

**噪声召回率 Se = 100%**——所有合成噪声段均被正确识别。U_E 值是干净 ECG（1.39）的 2-3 倍，区分度明显。

### 22.4 INCART 12 导联 Holter 结果

```
36 条记录（30min 短 Holter，采样率 257Hz，12 导联）
MLII 导联平均 U_E：
  mean = 1.287, std = 0.699
  > P90 阈值：6/36 条被标记为"含噪声"（16.7%）
```

6/36 被标记，说明这些 30min Holter 记录中约 1/6 包含严重噪声窗口。这与 INCART 数据集本身是"接近临床 Holter"采集、部分记录含运动伪影的预期一致。

### 22.5 总体性能 vs 论文

| 指标 | 本地实验 | 论文 PN-QRS | 差距 |
|------|---------|------------|------|
| Se（噪声召回率） | **100.0%** | — | — |
| Sp（ECG 特异性） | **89.0%** | **96.84%** | **-7.8pp** |
| Acc | **90.75%** | **96.84%** | **-6.1pp** |

**差距根因：阈值过低（P90）导致假阳性过高**

```
论文评测设计:
  - 论文使用 CPSC2019-ECG 集 (96.84%) + PPG + 心音 + 真实噪声
  - 论文 U_E 阈值 α=0.1（概率空间），等价于"几乎任何不确定性都报噪声"

我们的实验:
  - CPSC2019-Test P90 = 1.9967
  - 该阈值对应约 10% 的干净 ECG 被误标（假阳性）
  - 调高到 P95（U_E ≈ 2.3）可提升 Sp 约 5pp，但 Se 可能轻微下降
```

```
论文 FP=15, FN=9 在同等基数下：
  误检率 = 0.009%，漏检率 = 0.006%

绝对值差 3-15 倍，但 F1 只差 0.05pp——
因为 F1 对 0.1% 以下的误差不敏感。
```

### 23.4 差距集中在 3 条困难记录

| 记录 | FN | Se | 说明 |
|------|----|----|------|
| I31  | 62 | 98.07% | 极高噪声（U_E≈3.5），模型不确定性高 |
| I18  | 30 | 99.03% | Holter 噪声段（U_E≈2.8） |
| I36  | 16 | 99.59% | 边界截断残留 |
| 其余 66 条 | 36 | ≈99.99% | 几乎完美 |

I31 和 I18 正是噪声筛选实验中 U_E 最高的记录——模型对这些记录本就不确定，QRS 检测精度也因此下降，行为一致。

### 23.5 论文能达到 99.99% 的可能原因

- 论文训练数据可能包含 INCART（leave-one-out 协议）→ 我们跨数据集泛化做到 99.94% 已很强
- 论文使用完整 75 条记录（我们只有 69 条有 .atr 标注文件）
- 论文可能有更细的 NMS/去重参数调优

## 24. SQI 接入 Finetune 规划（2026-03-30）

### 24.1 SQI 是什么

**Signal Quality Index（信号质量指数）**，给每段 ECG 打一个"这段信号有多可靠"的分数。

PN-QRS 的 epistemic uncertainty U_E 天然就是一个 SQI：

```
U_E 低（≈1.0）→ 模型很有把握 → 信号质量好
U_E 高（≈3.0）→ 模型很困惑  → 信号质量差（噪声/运动伪影）
```

与传统 SQI 方法（kSQI、GbSQI）的区别：

| 方法 | 原理 | 缺点 |
|------|------|------|
| kSQI | 峰度（高峰度 = 有 QRS） | 对低振幅 ECG 失效 |
| GbSQI | 频谱比（QRS 能量集中在 5-50Hz） | 无法区分肌电和 QRS |
| **PN-QRS U_E** | 模型认知不确定性 | 需要运行神经网络 |

论文实测：PN-QRS 噪声筛选准确率 96.84%（ECG 集），远超 kSQI（83.23%）和 GbSQI（74.60%）。

### 24.2 为什么要在 Finetune 阶段用 SQI

Mamba 预训练用对比学习（contrastive learning），对噪声有特殊敏感性：

```
对比学习的目标：
  同一患者不同窗口 → 特征空间靠近（正样本对）
  不同患者/不同状态 → 特征空间远离（负样本对）

如果负样本中混入噪声窗口：
  噪声特征 → 随机、无结构
  → 模型学到"离噪声远" ≠ "离不同病理状态远"
  → 预训练特征被污染，下游 finetune AUROC 下降
```

MIMIC-IV 实测噪声比例 1.5%——虽然比例低，但 800K × 1.5% = **12,000 条噪声窗口**会持续出现在 negative pair 采样中。

### 24.3 三种接入方案

#### 方案 1：硬过滤（最简单，推理开销大）

```
DataLoader 读取窗口
    ↓
实时运行 PN-QRS → 计算 U_E
    ↓
U_E > 2.45 → 跳过，不进入 batch
    ↓
只用干净窗口做对比学习
```

- **优点**：实现简单，2 行代码
- **缺点**：每个训练窗口都要多跑一次 PN-QRS，**训练速度约降低 40-60%**
- **适用**：数据量小（<100K）时可接受

#### 方案 2：预计算 SQI 标签（推荐）

```
离线阶段（一次性，约 8-12 小时 GPU）：
    对 800K 条全部运行 PN-QRS
    → 保存每条记录的 mean_U_E 到 sqi_labels.json

训练阶段：
    DataLoader 加载窗口时，同时读取预存的 U_E
    → U_E > 2.45：跳过 或 loss_weight = 0
```

- **优点**：不增加训练时延迟，预计算只做一次
- **缺点**：需要 8-12 小时 GPU 预处理时间
- **推荐指数**：★★★★☆

#### 方案 3：软降权（最优，最复杂）

```python
# DataLoader 返回 (ecg_window, sqi_score)
weight = torch.exp(-sqi_score)  # U_E 越高，权重越低

# 加权对比损失
loss = weight * contrastive_loss(z_i, z_j)
loss.mean().backward()
```

- **优点**：不丢弃任何数据，平滑降权；低质量样本有弱贡献而非零贡献
- **缺点**：需要修改损失函数，调参 weight schedule
- **推荐指数**：★★★★★（如果有时间）

### 24.4 性价比分析

| 方案 | 预处理成本 | 训练额外成本 | 预期 AUROC 提升 | 推荐 |
|------|----------|------------|----------------|------|
| 不做（现状） | 0 | 0 | — | 数据干净时合理 |
| 方案 1 硬过滤（实时） | 0 | 训练慢 50% | +0.01-0.05pp | ❌ 太贵 |
| **方案 2 预计算过滤** | **8-12h GPU** | **0** | **+0.01-0.05pp** | **✅ 如果做** |
| 方案 3 软降权 | 8-12h GPU | 需改损失函数 | +0.02-0.08pp | ✅ 有余力时 |

### 24.5 关键判断：值不值得做？

**当前结论：优先级低，可以推迟。**

理由：
1. **MIMIC-IV 噪声仅 1.5%**：即使 12,000 条噪声全部清除，800K 里剩 788K，对比学习负样本池几乎不变
2. **对 AUROC 提升有限**：800K 预训练 vs SelfMIS（0.8255）已经是主要变量，SQI 是第二梯队优化
3. **当前主线堵塞**：Mamba 800K 预训练 OOM 还没修复，finetune 还没跑完 3-seed，SQI 是锦上添花

**什么时候应该做 SQI：**
- 如果用的是**自采上臂环数据**（噪声比例可能 >10%），SQI 价值大幅提升
- 如果 800K 预训练跑完后 AUROC 仍低于 0.8255（SelfMIS），且排除了其他原因，再考虑数据质量
- 如果有**实时推理场景**（用户佩戴设备时），SQI 是必须的安全门控

### 24.6 与当前主线的优先级对比

```
当前主线（按优先级）：
  P0  Mamba 800K OOM 修复 → 重启 epoch 13
  P0  800K 预训练跑完 → 3-seed finetune → 与 SelfMIS 对比
  P1  SQI 接入（如果 AUROC 仍不够）
  P2  PN-QRS R-peak 分割（等上臂环数据）
```

**上臂 ECG 专用数据集**：目前仍不存在开源版本。2024 ESC 大会有一篇摘要在上臂评测了 QRS 检测（98–99%），但数据未公开。

### 25.2 最接近的选项：MIT-BIH Noise Stress Test Database (NSTDB)

NSTDB 是专门为测试 QRS 检测器噪声鲁棒性设计的数据集：

- **来源**：取 MIT-BIH 干净 ECG，叠加真实录制的三类噪声
- **噪声类型**：
  - `bw`（baseline wander，基线漂移）：呼吸、体位变化引起的低频漂移
  - `ma`（muscle artifact，肌电伪影）：肌肉收缩引起的高频干扰，**上臂设备最主要噪声源**
  - `em`（electrode motion，电极运动）：运动导致电极接触变化，**上臂设备第二大噪声源**
- **SNR 档位**：24 / 18 / 12 / 6 / 0 / −6 dB 六档
- **标注**：完整继承 MIT-BIH 的 QRS beat 标注

`ma + em` 组合是上臂设备噪声特征的最佳公开代理。

### 25.3 加噪声模拟非标准导联：能做什么、不能做什么

#### 可以模拟的部分

| 噪声类型 | 对应上臂设备场景 | 模拟方法 |
|---------|--------------|---------|
| 基线漂移（bw） | 呼吸、手臂摆动 | 叠加 0.05–0.5Hz 随机游走或正弦波 |
| 肌电伪影（ma） | 上臂肌肉运动 | 叠加宽带高频噪声（20–500Hz） |
| 电极运动（em） | 设备松动、皮肤滑动 | 叠加低频脉冲式基线跳变 |
| 幅度衰减 | 上臂离心脏远，QRS 幅度约降至 1/3–1/2 | 信号 × 0.3–0.5 缩放 |

#### **不能**靠加噪声解决的部分

```
QRS 形态变化（根本问题）：
  标准导联（MLII）：心电轴接近额面，QRS 高大直立
  上臂导联：心电轴不同，QRS 可能矮小、双相甚至倒置
  → 这是信号本身的问题，加噪声改变不了波形形状
```

加噪声只能测试模型的**噪声鲁棒性**，不能替代真实上臂数据的形态学测试。

### 25.4 可操作的代理测试方案

利用 LTDB + NSTDB 噪声构造"最接近上臂 Holter 场景"的开源代理测试集：

```
Step 1：取 LTDB（单导联，128Hz，24h，最接近低采样率单导联场景）
Step 2：叠加 NSTDB 录制的真实 ma（肌电）噪声
Step 3：幅度缩放 × 0.4（模拟上臂信号衰减）
Step 4：在不同 SNR 下（24/12/6/0 dB）测试 PN-QRS F1

预期结论：
  - 若在 6 dB SNR 下 F1 仍 > 98%，则 PN-QRS 对上臂设备有参考价值
  - 若在 0 dB SNR 下 F1 明显下降，需要在 DataLoader 中加 SQI 门控
```

所有数据均在 PhysioNet 开源、wfdb 可直接下载，无需授权。

### 25.5 与 PN-QRS 论文的 NSTDB 实验对比

PN-QRS 论文已经在 NSTDB 上做了噪声筛选实验（Table IV），但只评测了**筛选能力**（Se/Sp），没有直接报告加噪后的 QRS 检测 F1。本地复现结果（Section 22）：

| 噪声类型 | U_E 阈值 t=2.45 | Se（噪声召回） | Sp（干净信号保留） |
|---------|---------------|------------|----------------|
| em（电极运动） | 2.45 | 100% | 89% |
| bw + ma 组合 | 2.45 | 100% | 89% |

**漏洞**：Sp=89% 意味着 11% 的干净窗口被误判为噪声丢弃，在 QRS 检测流程中会产生漏检（FN）。这是在上臂设备场景下接入 PN-QRS 噪声筛选时需要关注的主要问题。

### 25.6 优先级建议

```
短期（已有数据可做）：
  在 LTDB 全量结果出来后，叠加 NSTDB 的 ma 噪声，
  测试 SNR 扫描下的 F1 曲线（约 2–3 小时实验）

中期（需要资源）：
  用上臂设备自采一批带标注数据（哪怕 10–30 分钟），
  才能真正验证形态学差异对检测率的影响

长期（项目深化）：
  形态域迁移（domain adaptation）——从标准导联形态
  迁移到上臂导联形态，类似跨域 QRS 训练
```

**结论：先把 Mamba 800K 跑完，再根据结果决定是否需要 SQI。**

### 26.1 为什么需要更难的测试集？

```
模型 A（真正鲁棒）：MIT-BIH 99.9%  →  极端场景 98.5%  → 差 1.4pp
模型 B（表面漂亮）：MIT-BIH 99.8%  →  极端场景 92.0%  → 差 7.8pp
```

区分度来自**数据集本身的难度梯度**，不是数据量。

## 27. apply_pnqrs.py 修改记录与标注数据集回归验证（2026-04-08）

> 背景：为支持自采设备数据（ADC 计数格式），对 PN-QRS 推理流程做了适配，本节记录修改内容及对原有测试集的影响验证。

### 27.1 修改内容

**原始代码（论文版本）**：评测脚本（`evaluate_mitbih_v2.py` 等）直接调用 `preprocess_ecg(window, fs=fs)`，信号单位默认为 mV。

**新增文件 `apply_pnqrs.py`**：自采数据入口，在 `_run_window` 中加入预归一化：

```python
std = window_1d.std()
if std > 1:   # mV 量级信号 std 通常 < 1，不触发
    window_1d = (window_1d - window_1d.mean()) / std
```

**修改原因**：`preprocess_ecg` 内部的 `pp()` 函数设计用于 mV 信号：

```python
def pp(data):
    if np.max(abs(data)) > 10:          # 阈值 10mV
        b = np.argwhere(abs(data) > 10)
        for k in b[:,0]:
            if k > 0:
                data[k] = data[k-1]     # 超阈值样本替换为前一个值
```

自采设备输出 ADC 计数（幅度 ±5000），`pp()` 会把**整段信号抹平成常数**，导致模型输入为全零，检测完全失败。预 z-score 将信号幅度压缩到 ±3 左右，使 `pp()` 不再触发。

**原始代码文件未做任何修改**（`dataset/`、`models/`、`utils/` 目录保持原样）。

### 27.2 对原有标注数据集的影响验证

验证思路：标准 ECG 信号（mV 单位）最大幅度远低于 10mV，`pp()` 从未触发，因此预 z-score 是否执行在数学上等价。

**实测验证（LTDB 14046，std 最大的窗口）**：

| 项目 | 值 |
|------|-----|
| 窗口 std | 1.121 mV |
| 窗口 max\_abs | 5.110 mV |
| pp() 阈值 | 10 mV |
| pp() 是否触发 | **否** |
| 有无预 z-score 的输出最大差值 | **4.77e-07**（浮点精度，实质相同） |

**数学解释**：带通滤波是线性运算，z-score 是线性缩放。  
`z-score(bandpass(z-score(x)))` = `z-score(bandpass(x))`，两路输出等价。
