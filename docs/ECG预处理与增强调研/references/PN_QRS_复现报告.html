Section 00

背景与技术原理

痛点

为什么现有 QRS 检测器不够用？

医院 ECG 信号干净，传统方法（Pan-Tompkins）够用。
但可穿戴设备采集长程心电时：
  • 运动伪差、基线漂移、电极接触不良
  • 噪声段看起来像 QRS，检测器无法识别
  • 传统方法只给"是/否"，不告诉你它有多确信
创新点

PN-QRS 的核心回答

论文名 Positive-Negative 来自其核心技术：
对每一帧信号创造"乐观分身"（φ⁺）和"悲观分身"（φ⁻），
通过两个分身的分歧程度估计不确定性，
从而实现三层输出：
● 这段信号是噪声（整段丢弃） ● 这个心跳形态模糊（标记可疑） ● R 峰精确位置（置信输出）

两种不确定性：系统的核心区分

UE
Epistemic 认知不确定性
"模型没见过这类信号"
训练数据只有 ECG。当输入 PPG、心音或纯噪声时， 正负分身预测结果分歧巨大 → U_E 飙升。

U_E 高 → 整段 10s 窗口是非 ECG，直接丢弃。
计算方式
UE = KL(ŷ⁺ ‖ ŷ) + KL(ŷ⁻ ‖ ŷ)
阈值 α = 0.1（论文）/ 2.45（本地模型标定）
实测：纯噪声 U_E ≈ 3–4，干净 ECG ≈ 1.4，完全可分
UA
Aleatoric 偶然不确定性
"数据本身就模糊，谁来看都犹豫"
信号本身是 ECG，但该位置的心跳形态特殊（室早、束支阻滞）， 即使是专家也会反复确认。

U_A 高 → 该 R 峰被标记为可疑心跳，信号本身保留。
计算方式
UA = −[ŷ⁺ log ŷ⁺ + (1−ŷ⁻) log(1−ŷ⁻)]
阈值 β = 0.12，取 R 峰前后 7 帧均值
区别 U_E：不丢弃整段，只标记该心跳，后续分析可选择是否使用

Section 02.5

数据处理流程：从原始 WFDB 到模型输入

各数据集均为 WFDB 原始格式（.dat + .hea + .atr）， 采样率、记录长度差异大。PN-QRS 模型工作在 50 Hz 内部空间，所有数据需经统一预处理后才能送入推理。

数据集采样率记录长度导联数每窗口采样点主要挑战
